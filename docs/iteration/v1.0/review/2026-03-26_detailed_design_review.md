## 二、v1.2 新发现问题

### 🟡 新增问题

#### 1. 章节 4.3 - ComplexityEngine 类接口缺少配置参数

**问题描述**：

章节 1.2.2 定义了四个维度的复杂度评分权重（文件数 30%、技术栈 20%、依赖深度 30%、历史成功率 20%），但章节 4.3.1 的 `ComplexityEngine` 类接口没有体现这些权重配置：

```python
class ComplexityEngine:
    def __init__(self, config: Optional[ComplexityConfig] = None):
        self.config = config or ComplexityConfig()
```

但 `ComplexityConfig` 类在文档中并未定义。

**影响**：实现者无法从文档中得知如何配置权重。

**建议**：

在章节 4.3.1 补充 `ComplexityConfig` 数据类定义：

```python
@dataclass
class ComplexityConfig:
    """复杂度评估配置"""
    file_count_weight: float = 0.3
    tech_stack_weight: float = 0.2
    dependency_depth_weight: float = 0.3
    historical_success_weight: float = 0.2

    # 阈值配置
    simple_threshold: float = 40.0
    medium_threshold: float = 70.0
```

**优先级**: 💡 P3（低优先级，实现时可自行推导）

---

#### 2. 章节 3.2.1 - 多模型自动降级缺少模型映射表

**问题描述**：

第 803-814 行的多模型自动降级流程提到"降级到备用模型 1（如 Claude 3 Haiku）"，但没有定义：

1. 如何配置主模型和备用模型的映射关系
2. 降级后的模型能力差异如何处理（如 Haiku 不支持某些工具）

**建议**：

补充模型能力映射表：

```markdown
**模型能力矩阵**：

| 模型 | Tool Use | Vision | Max Tokens |
|------|----------|--------|------------|
| claude-3-5-sonnet | ✅ | ✅ | 200K |
| claude-3-haiku | ✅ | ✅ | 200K |
| gpt-4 | ✅ | ❌ | 128K |
| gpt-3.5-turbo | ✅ | ❌ | 16K |

**降级时的能力处理**：
- 如果备用模型不支持某工具，标记为"不可用"
- 向用户提示功能可能受限
```

**优先级**: 💡 P3（低优先级）

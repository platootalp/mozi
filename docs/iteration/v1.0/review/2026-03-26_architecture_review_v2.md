## 二、问题分类与修改建议

### 🔴 严重问题（逻辑错误）

#### 1. 第2.3节 - 配置加载优先级方向错误

**当前描述**（第149-153行）：
```markdown
配置加载优先级（从高到低）：

1. 环境变量 (`MOZI_*`)
2. user.json (用户全局)
3. .mozi/*.json (项目本地)
4. 系统默认配置
```

**问题分析**：
- 描述为"从高到低"，但列出的顺序不符合常规配置优先级逻辑
- 通常**越具体的配置越优先**：项目本地 > 用户全局 > 系统默认
- 环境变量通常用于临时覆盖，优先级最高是合理的

**修改方案**（二选一）：

**方案A - 调整顺序**（推荐）：
```markdown
配置加载优先级（从高到低）：

1. 环境变量 (`MOZI_*`) —— 用于临时覆盖
2. .mozi/*.json (项目本地) —— 项目特定配置
3. user.json (用户全局) —— 用户偏好
4. 系统默认配置 —— 兜底默认值
```

**方案B - 修改描述**：
```markdown
配置加载顺序（后加载覆盖前者）：
1. 系统默认配置
2. user.json (用户全局)
3. .mozi/*.json (项目本地)
4. 环境变量 (`MOZI_*`)
```

### 🟠 中等问题（机制描述不完整）

#### 2. 第4.3节 - 哈希锚定冲突处理缺失

**当前描述**（第340行）：
> 编辑前校验锚点是否匹配，不匹配则返回冲突信息。

**问题**：仅"返回冲突信息"过于简略，未说明后续处理流程。

**建议补充**：
```markdown
**哈希锚定编辑机制**：

解决多Agent并发编辑冲突，每处编辑包含：
- 行号标识（LINE#ID）
- 原始内容哈希
- 期望替换内容

**冲突处理策略**：
1. **首次冲突** → 自动重试（重新读取文件最新状态，重新计算哈希）
2. **连续冲突**（重试3次失败）→ 向用户展示冲突详情，等待决策
3. **涉及关键文件**（如.gitignore、配置文件）→ 强制转为HITL审批模式
```

#### 3. 第6.1节 - Agent崩溃checkpoint机制缺失

**当前描述**（第395行）：
> Agent崩溃 → 会话恢复+状态回滚 → 从上一个checkpoint恢复

**问题**：未说明checkpoint何时创建、保留策略等。

**建议补充**：
```markdown
**Agent崩溃恢复**：

- 会话恢复+状态回滚
- 从上一个checkpoint恢复

**Checkpoint策略**：
- 创建时机：每完成一个工具调用后、每个Agent步骤完成后、显式检查点
- 存储内容：会话上下文、工具状态、执行位置标记、变量绑定
- 保留策略：保留最近10个checkpoint，自动清理超过7天的旧数据
- 恢复粒度：可选择回滚到最近checkpoint或完全重启会话
```

#### 4. 第8.1节 - SQLite并发优化说明缺失

**问题**：操作日志和性能追踪都写入SQLite（7.1节），高并发场景可能成为瓶颈，但未说明优化措施。

**建议补充**（在第8.1节"长期记忆"行后或新增段落）：
```markdown
**SQLite优化策略**：
- WAL模式（Write-Ahead Logging）支持并发读写
- 批量写入缓冲（100ms/1000条批量提交）
- 大表自动分区（按日期分区操作日志表）
- 读写分离：查询走副本，写入走主库
```

## 三、可选优化建议

### 5. 第3层"能力层"内容过载

**当前问题**：能力层包含配置管理、工具框架、MCP接入层、Skills引擎、安全与权限五个子模块，内容密度较高。

**可选方案**（非必须修改）：

```
方案A - 保持现状（推荐，改动成本较高）
方案B - 将"配置管理"上浮为独立层
方案C - 配置管理并入"编排层"（配置是调度的一部分）
```

若选择方案B，架构可调整为：
```
接入层 → 配置层 → 编排层 → 能力层 → 基础设施层
```

### 6. 缺少系统启动流程说明

**建议新增一节** `11. 系统启动流程`：

```markdown
## 11. 系统启动流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 1. 加载配置 │ → │ 2. 初始化   │ → │ 3. 连接服务 │ → │ 4. 就绪检查 │
│             │    │    存储层   │    │  (MCP/模型) │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

**详细步骤**：
1. **配置加载**（10ms级）
   - 按优先级加载 config.json / agents.json / tools.json / mcp.json
   - 验证JSON Schema，错误时优雅降级

2. **存储层初始化**（100ms级）
   - SQLite连接池建立（WAL模式启用）
   - 向量库连接（本地优先，云端回退）
   - 归档目录检查

3. **服务连接**（1-5s级，可异步）
   - MCP Server健康检查与能力发现
   - 模型API连通性测试
   - 失败时标记为不可用，不阻断启动

4. **就绪检查**
   - /health端点可用
   - 日志系统就绪
   - 接受用户请求
```

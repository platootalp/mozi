## 1. 模块概述

Ingress 模块是系统的入口层，负责接收用户输入并返回系统响应。支持三种接入方式：CLI 命令行、Web UI、IDE 插件。

### 1.1 核心职责

- 命令解析与参数验证
- 用户会话管理
- 响应渲染与格式化
- 认证与授权（V2.0）

### 1.2 差异化特性

| 特性 | 说明 |
|------|------|
| 多端一致 | CLI/Web/IDE 体验一致 |
| 流式输出 | 支持 Server-Sent Events |
| 命令补全 | 支持 Tab 自动补全 |
| 多会话管理 | 支持并发会话 |

## 3. 组件详情

### 3.1 CLI (Typer)

**职责**: 命令行交互入口

**核心功能点**:

| 功能 | 说明 |
|------|------|
| `mozi` | 启动交互式会话 |
| `mozi "task"` | 带初始提示的会话 |
| `mozi -p "query"` | 非交互模式（单次查询） |
| `mozi -c` | 继续最近会话 |
| `mozi -r <session>` | 恢复指定会话 |
| `mozi --config <path>` | 指定配置文件 |
| `mozi --model <model>` | 指定模型 |
| `mozi list` | 列出所有会话 |
| `mozi delete <session>` | 删除会话 |

**命令选项**:

```python
# mozi 命令
@cli.command()
def main(
    task: Optional[str] = typer.Option(None, help="初始任务描述"),
    interactive: bool = typer.Option(True, "-p", help="非交互模式"),
    continue_last: bool = typer.Option(False, "-c", help="继续最近会话"),
    resume: Optional[str] = typer.Option(None, "-r", help="恢复指定会话"),
    model: Optional[str] = typer.Option(None, "--model", help="指定模型"),
    config: Optional[Path] = typer.Option(None, "--config", help="配置文件路径"),
):
    pass
```

### 3.2 Web UI (FastAPI)

**职责**: REST API 和 WebSocket 通信

**API 端点**:

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/sessions` | 创建会话 |
| GET | `/api/v1/sessions` | 列出会话 |
| GET | `/api/v1/sessions/{id}` | 获取会话详情 |
| DELETE | `/api/v1/sessions/{id}` | 删除会话 |
| POST | `/api/v1/sessions/{id}/tasks` | 创建任务 |
| GET | `/api/v1/tasks/{id}` | 获取任务状态 |
| WS | `/ws/{session_id}` | WebSocket 实时通信 |

**功能点**:

## 4. 核心工作流

### 4.1 CLI 交互流程

```
用户输入
    │
    ▼
┌─────────────────┐
│  Typer CLI     │ ← 命令解析 & 参数验证
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  SessionManager │ ← 会话创建/恢复
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Orchestrator   │ → 转发到编排层
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Renderer       │ ← 响应格式化
└────────┬────────┘
         │
         ▼
终端输出 / JSON
```

### 4.2 WebSocket 实时通信流程

```
客户端
    │
    ├──→ WS Connect ──→ SessionManager.register()
    │
    ├──→ Send Message ──→ Orchestrator.execute()
    │                        │
    │                        ▼
    │                    Agent 处理中...
    │                        │
    │←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←┘
    │
    └──→ Disconnect ──→ SessionManager.unregister()
```

## 5. 会话管理

### 5.1 会话状态

| 状态 | 说明 |
|------|------|
| `ACTIVE` | 活跃会话 |
| `IDLE` | 空闲会话（超时未活动） |
| `COMPLETED` | 已完成 |
| `ERROR` | 错误状态 |

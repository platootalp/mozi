## 1. 模块概述

Model 模块是系统的模型网关层，负责统一管理多个 LLM Provider。通过抽象接口，支持 Anthropic、OpenAI、Ollama 等多种模型，提供统一的调用方式和智能路由能力。

### 1.1 核心职责

- 多模型统一接口
- 模型智能路由
- Provider 适配
- 请求/响应处理
- Token 计费管理

### 1.2 差异化特性

| 特性 | 说明 |
|------|------|
| 多 Provider | 支持 Anthropic/OpenAI/Ollama |
| 统一接口 | 屏蔽 Provider 差异 |
| 智能路由 | 根据任务类型选择最佳模型 |
| 成本优化 | Token 用量追踪与控制 |

## 3. 组件详情

### 3.1 ModelGateway

**职责**: 多模型统一入口

**核心功能点**:

| 功能 | 说明 |
|------|------|
| complete() | 发送对话请求 |
| embeddings() | 获取文本嵌入 |
| 模型路由 | 根据配置/任务选择模型 |
| 响应缓存 | 缓存相同请求的响应 |

**接口定义**:

```python
class ModelGateway:
    PROVIDERS: Dict[str, Type[BaseProvider]] = {
        "anthropic": AnthropicProvider,
        "openai": OpenAIProvider,
        "ollama": OllamaProvider,
    }

    def __init__(
        self,
        router: Optional[Router] = None,
        cache: Optional[ResponseCache] = None,
    ):
        self.router = router or DefaultRouter()
        self.cache = cache or ResponseCache()
        self.providers: Dict[str, BaseProvider] = {}
        self._init_providers()

    def _init_providers(self) -> None:
        """初始化所有 Provider"""
        for name, provider_class in self.PROVIDERS.items():
            self.providers[name] = provider_class()

    async def complete(
        self,
        messages: List[Message],
        model: Optional[str] = None,
        **kwargs
    ) -> Response:
        """发送对话请求"""
        # 1. 解析模型名称
        provider_name, model_name = self.router.resolve(model)

        # 2. 检查缓存
        cache_key = self._make_cache_key(messages, model_name)
        if cached := await self.cache.get(cache_key):
            return cached

        # 3. 调用 Provider
        provider = self.providers[provider_name]
        response = await provider.complete(messages, model_name, **kwargs)

        # 4. 缓存响应
        await self.cache.set(cache_key, response)

        return response

    async def embeddings(self, texts: List[str], model: Optional[str] = None) -> List[float]:
        """获取文本嵌入"""
        provider_name, model_name = self.router.resolve(model or "embed")
        provider = self.providers.get(provider_name)
        if not provider:
            raise ModelNotFoundError(f"Provider {provider_name} not found")
        return await provider.embeddings(texts, model_name)
```

### 3.2 Router

**职责**: 模型路由决策

**路由策略**:

| 策略 | 说明 |
|------|------|
| 默认路由 | 使用配置中的默认模型 |
| 任务路由 | 根据任务类型选择模型 |
| 成本路由 | 选择最低成本模型 |
| 负载路由 | 均衡多个模型负载 |

```python
class Router(ABC):
    @abstractmethod
    def resolve(self, model: Optional[str]) -> Tuple[str, str]:
        """
        解析模型名称
        返回: (provider_name, model_name)
        """
        pass

class DefaultRouter(Router):
    def __init__(self, default_provider: str = "anthropic", default_model: str = "claude-sonnet-4-20250514"):
        self.default_provider = default_provider
        self.default_model = default_model

    def resolve(self, model: Optional[str]) -> Tuple[str, str]:
        if model:
            return self._parse_model(model)
        return self.default_provider, self.default_model

    def _parse_model(self, model: str) -> Tuple[str, str]:
        """解析 model 名称，支持 'provider/model' 格式"""
        if "/" in model:
            provider, name = model.split("/", 1)
            return provider, name
        return self.default_provider, model

class TaskRouter(Router):
    """基于任务类型的路由"""
    TASK_MODEL_MAP = {
        Intent.CODE: ("anthropic", "claude-sonnet-4-20250514"),
        Intent.REVIEW: ("anthropic", "claude-sonnet-4-20250514"),
        Intent.EXPLORE: ("openai", "gpt-4o"),
        Intent.PLAN: ("anthropic", "claude-opus-4-6"),
        Intent.RESEARCH: ("openai", "gpt-4o"),
    }

    def resolve(self, model: Optional[str]) -> Tuple[str, str]:
        if model:
            return self._parse_model(model)

        # 根据上下文中的意图选择模型
        intent = self._get_current_intent()
        return self.TASK_MODEL_MAP.get(intent, ("anthropic", "claude-sonnet-4-20250514"))
```

### 3.3 BaseProvider

**职责**: Provider 抽象基类

```python
class BaseProvider(ABC):
    """Provider 基类，所有 Provider 必须实现"""

    @property
    @abstractmethod
    def name(self) -> str:
        """Provider 名称"""
        pass

    @property
    @abstractmethod
    def supported_models(self) -> List[str]:
        """支持的模型列表"""
        pass

    @abstractmethod
    async def complete(
        self,
        messages: List[Message],
        model: str,
        **kwargs
    ) -> Response:
        """发送对话请求"""
        pass

    @abstractmethod
    async def embeddings(self, texts: List[str], model: str) -> List[float]:
        """获取文本嵌入"""
        pass

    def validate_model(self, model: str) -> bool:
        """验证模型是否支持"""
        return model in self.supported_models
```

### 3.4 AnthropicProvider

**职责**: Anthropic API 适配

**Message 格式差异说明**:

Anthropic API 使用 `role` 和 `content` 字段，消息格式与 OpenAI 略有不同：
- 支持 `user` 和 `assistant` 角色
- 不支持 `system` 角色，系统消息需作为首条 `user` 消息传入

```python
class AnthropicProvider(BaseProvider):
    BASE_URL = "https://api.anthropic.com/v1"

    def __init__(self, api_key: Optional[str] = None):
        self.api_key = api_key or os.getenv("ANTHROPIC_API_KEY")
        self.client = Anthropic(api_key=self.api_key)

    @property
    def name(self) -> str:
        return "anthropic"

    @property
    def supported_models(self) -> List[str]:
        return [
            "claude-opus-4-6",
            "claude-sonnet-4-20250514",
            "claude-haiku-4-5-20251001",
        ]

    def _to_anthropic_format(self, messages: List[Message]) -> List[Dict[str, str]]:
        """
        将统一 Message 格式转换为 Anthropic 格式

        Anthropic 要求：
        - system 消息需合并到首条 user 消息中
        - 消息必须以 user 或 assistant 开头
        """
        anthropic_messages = []
        system_content = []

        for msg in messages:
            if msg.role == "system":
                system_content.append(msg.content)
            else:
                anthropic_messages.append({"role": msg.role, "content": msg.content})

        # 将 system 内容注入到首条 user 消息
        if system_content and anthropic_messages and anthropic_messages[0]["role"] == "user":
            anthropic_messages[0]["content"] = "\n\n".join(system_content + [anthropic_messages[0]["content"]])
        elif system_content:
            # 如果首条不是 user，插入一条 user 消息作为 system
            anthropic_messages.insert(0, {"role": "user", "content": "\n\n".join(system_content)})

        return anthropic_messages

    async def complete(
        self,
        messages: List[Message],
        model: str,
        **kwargs
    ) -> Response:
        """调用 Anthropic API"""
        anthropic_messages = self._to_anthropic_format(messages)
        response = await self.client.messages.create(
            model=model,
            messages=anthropic_messages,
            **kwargs
        )
        return Response(
            content=response.content[0].text,
            model=model,
            usage=Usage(
                input_tokens=response.usage.input_tokens,
                output_tokens=response.usage.output_tokens,
            ),
            stop_reason=response.stop_reason,
        )

    async def embeddings(self, texts: List[str], model: str) -> List[float]:
        raise NotImplementedError("Anthropic does not support embeddings")
```

### 3.5 OpenAIProvider

**职责**: OpenAI API 适配

**Message 格式差异说明**:

OpenAI Chat API 使用标准 `role`/`content` 结构：
- 支持 `system`、`user`、`assistant` 角色
- `system` 角色作为独立消息传递

```python
class OpenAIProvider(BaseProvider):
    BASE_URL = "https://api.openai.com/v1"

    def __init__(self, api_key: Optional[str] = None):
        self.api_key = api_key or os.getenv("OPENAI_API_KEY")
        self.client = OpenAI(api_key=self.api_key)

    @property
    def name(self) -> str:
        return "openai"

    @property
    def supported_models(self) -> List[str]:
        return [
            "gpt-4o",
            "gpt-4o-mini",
            "gpt-4-turbo",
            "gpt-3.5-turbo",
        ]

    def _to_openai_format(self, messages: List[Message]) -> List[Dict[str, str]]:
        """
        将统一 Message 格式转换为 OpenAI 格式

        OpenAI 要求：
        - system 消息作为独立消息存在
        - 消息必须以 user 或 assistant 开头
        - 不允许连续两条相同角色的消息
        """
        openai_messages = []
        last_role = None

        for msg in messages:
            # 合并连续相同角色的消息
            if msg.role == last_role and openai_messages:
                openai_messages[-1]["content"] += "\n" + msg.content
            else:
                openai_messages.append({"role": msg.role, "content": msg.content})
                last_role = msg.role

        return openai_messages

    async def complete(
        self,
        messages: List[Message],
        model: str,
        **kwargs
    ) -> Response:
        """调用 OpenAI API"""
        openai_messages = self._to_openai_format(messages)
        response = await self.client.chat.completions.create(
            model=model,
            messages=openai_messages,
            **kwargs
        )
        return Response(
            content=response.choices[0].message.content,
            model=model,
            usage=Usage(
                input_tokens=response.usage.prompt_tokens,
                output_tokens=response.usage.completion_tokens,
            ),
            stop_reason=response.choices[0].finish_reason,
        )

    async def embeddings(self, texts: List[str], model: str = "text-embedding-3-small") -> List[float]:
        """获取文本嵌入"""
        response = await self.client.embeddings.create(
            model=model,
            input=texts,
        )
        return [item.embedding for item in response.data]
```

### 3.6 OllamaProvider

**职责**: Ollama 本地模型适配

**依赖说明**:
```python
# 需要安装 httpx: uv add httpx
from httpx import AsyncClient as AsyncHTTPClient
```

```python
class OllamaProvider(BaseProvider):
    BASE_URL = "http://localhost:11434"

    def __init__(self, base_url: Optional[str] = None):
        self.base_url = base_url or os.getenv("OLLAMA_BASE_URL", self.BASE_URL)
        # 需要安装 httpx: uv add httpx
        from httpx import AsyncClient as AsyncHTTPClient
        self.client = AsyncHTTPClient(base_url=self.base_url)

    @property
    def name(self) -> str:
        return "ollama"

    @property
    def supported_models(self) -> List[str]:
        return ["llama2", "codellama", "mistral", "mixtral"]

    async def complete(
        self,
        messages: List[Message],
        model: str,
        **kwargs
    ) -> Response:
        """调用 Ollama API"""
        payload = {
            "model": model,
            "messages": [m.to_dict() for m in messages],
            **kwargs
        }
        response = await self.client.post("/api/chat", json=payload)
        data = response.json()
        return Response(
            content=data["message"]["content"],
            model=model,
            usage=Usage(
                input_tokens=data.get("prompt_eval_count", 0),
                output_tokens=data.get("eval_count", 0),
            )
        )

    async def embeddings(self, texts: List[str], model: str = "nomic-embed-text") -> List[float]:
        """获取文本嵌入"""
        embeddings = []
        for text in texts:
            payload = {"model": model, "prompt": text}
            response = await self.client.post("/api/embeddings", json=payload)
            embeddings.append(response.json()["embedding"])
        return embeddings
```

### 3.7 Provider Message 格式差异

不同 Provider 的 Message 格式存在差异，Model 模块通过格式转换方法处理：

| Provider | 角色支持 | System 消息处理 | 连续相同角色 |
|----------|---------|----------------|-------------|
| Anthropic | user, assistant | 合并到首条 user 消息 | 需要合并 |
| OpenAI | system, user, assistant | 独立消息传递 | 需要合并 |
| Ollama | system, user, assistant | 独立消息传递 | 无需合并 |

**转换策略**:

1. **Anthropic**: 不支持 `system` 角色，系统消息需合并到首条 `user` 消息中
2. **OpenAI**: 支持 `system` 角色，但需合并连续相同角色的消息
3. **Ollama**: 与 OpenAI 类似，但 API 层面已处理

**统一格式**:
```python
@dataclass
class Message:
    role: str      # "system" | "user" | "assistant"
    content: str
```

## 4. 响应缓存

```python
class ResponseCache:
    """响应缓存，支持 LRU 策略"""

    def __init__(self, max_size: int = 1000, ttl: int = 3600):
        self.max_size = max_size
        self.ttl = ttl
        self._cache: OrderedDict[str, CacheEntry] = OrderedDict()

    async def get(self, key: str) -> Optional[Response]:
        """获取缓存的响应"""
        if key not in self._cache:
            return None

        entry = self._cache[key]
        if time.time() - entry.created_at > self.ttl:
            del self._cache[key]
            return None

        # 移动到末尾（最近使用）
        self._cache.move_to_end(key)
        return entry.response

    async def set(self, key: str, response: Response) -> None:
        """设置缓存"""
        if len(self._cache) >= self.max_size:
            # 删除最旧的条目
            self._cache.popitem(last=False)

        self._cache[key] = CacheEntry(response=response, created_at=time.time())
```

## 7. Claude Code Model Alignment

本节说明 Mozi Model 模块与 Claude Code 模型策略的差异与对齐考量。

### 7.1 架构定位差异

| 维度 | Claude Code | Mozi |
|------|-------------|------|
| Provider 支持 | Anthropic only | 多 Provider（Anthropic/OpenAI/Ollama） |
| 模型选择 | 用户偏好设置 | 智能路由 + 配置 |
| 路由机制 | 隐式根据任务复杂度 | 显式 TaskRouter + 复杂度评估 |
| 目标用户 | 开发者个人 | 企业级多团队 |

**关键差异**: Claude Code 是面向开发者的单用户工具，模型选择是用户个人偏好。Mozi 是面向企业级场景的 AI 编码 Agent，模型路由是架构决策，涉及成本控制、合规审计、多团队资源共享等企业需求。

### 7.2 Sonnet 作为推荐默认

Claude Code 推荐 Sonnet 作为大多数任务的默认模型，理由如下：

- **性价比平衡**: Sonnet 在能力与成本之间取得良好平衡
- **代码能力**: 在编码任务上表现优异，成本低于 Opus
- **响应速度**: 相比 Opus 有更快的响应时间

Mozi 的默认模型同样采用 Sonnet（`claude-sonnet-4-20250514`），与 Claude Code 保持一致。

### 7.3 模型选择复杂度对照表

基于 Mozi 的复杂度评估体系，模型选择建议如下：

| 复杂度 | 分数范围 | 推荐模型 | 说明 |
|--------|----------|----------|------|
| SIMPLE | <=40 | `claude-haiku-4-5-20251001` | 快速轻量任务，如简单编辑、文件搜索 |
| MEDIUM | 40-70 | `claude-sonnet-4-20250514` | 标准任务，如代码生成、调试、代码审查 |
| COMPLEX | >70 | `claude-opus-4-6` | 复杂任务，如架构设计、多文件重构、深度代码分析 |

### 7.4 复杂度-模型选择启发

以下启发式规则可用于自动模型选择：

```
IF 任务类型 == REPL_TODO AND 复杂度 <= 40:
    推荐: claude-haiku-4-5-20251001
    理由: 快速响应，资源消耗低

ELIF 任务类型 == CODE_GENERATION AND 复杂度 <= 70:
    推荐: claude-sonnet-4-20250514
    理由: 优秀代码能力，性价比高

ELIF 任务类型 == ARCHITECTURE_DESIGN OR 复杂度 > 70:
    推荐: claude-opus-4-6
    理由: 深度推理能力，处理复杂逻辑

ELIF 用户明确指定模型:
    推荐: 用户选择
    理由: 尊重用户偏好
```

### 7.5 与 Claude Code 的互操作性

- **工具调用兼容**: Claude Code 与 Mozi 使用相同的工具定义格式（read、edit、bash 等），模型输出格式兼容
- **Context 共享**: 两者均可访问相同的代码库 Context
- **Session 互通**: Mozi 支持导入 Claude Code 的对话历史（通过 session 管理模块）

### 7.6 架构决策建议

1. **保持多 Provider 架构**: 企业场景需要 OpenAI/Ollama 作为备选或补充
2. **显式路由优于隐式**: Mozi 的复杂度评估 + 路由机制是企业合规所需
3. **默认对齐 Claude Code**: 默认模型与 Sonnet 保持一致，降低用户迁移成本
4. **支持用户覆盖**: 允许用户在配置或任务中显式指定模型

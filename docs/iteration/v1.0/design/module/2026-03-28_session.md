## 1. 模块概述

Session 模块是系统的会话管理层，负责管理用户与 Agent 之间的对话会话。包括会话创建、恢复、持久化，以及长对话的上下文压缩功能。

### 1.1 核心职责

- 会话生命周期管理（创建、暂停、恢复、完成）
- 会话持久化（SQLite 元数据 + JSON Lines 对话历史）
- 上下文压缩（防止 context overflow）
- 命名会话支持（通过名称查找和恢复）
- CLI 会话控制（-c/-r/-n/-p 标志）

### 1.2 差异化特性

| 特性 | 说明 |
|------|------|
| 混合存储 | SQLite 存储元数据，JSON Lines 存储对话历史 |
| 上下文压缩 | 95% context 容量时自动压缩历史消息 |
| 命名会话 | 支持通过名称查找和恢复会话 |
| 状态机管理 | 完整的状态转换：ACTIVE ↔ PAUSED → COMPLETED/ABANDONED/ERROR |

```python
class SessionState(Enum):
```

### 3.2 SessionMessage (orchestrator/session/models.py)

单条对话消息：

```python
@dataclass
```

### 3.3 Session (storage/session/schema.py)

会话元数据（存储在 SQLite）：

```python
@dataclass
class Session:
    """会话元数据"""
    id: str                         # 会话 ID
    created_at: datetime             # 创建时间
    updated_at: datetime             # 更新时间
    status: SessionStatus           # 状态
    complexity_level: ComplexityLevel | None = None  # 复杂度等级
    complexity_score: int | None = None              # 复杂度分数
    model: str | None = None        # 使用的模型
    message_count: int = 0          # 消息计数
    metadata: dict[str, Any] = field(default_factory=dict)  # 元数据
    last_activity: datetime | None = None           # 最后活动时间
    name: str | None = None         # 命名会话名称
```

## 4. 存储结构

### 4.1 SQLite 表 (storage/session/schema.py)

```sql
CREATE TABLE IF NOT EXISTS sessions (
    id TEXT PRIMARY KEY,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    status TEXT NOT NULL,
    complexity_level TEXT,
    complexity_score INTEGER,
    model TEXT,
    message_count INTEGER DEFAULT 0,
    metadata JSON,
    last_activity TIMESTAMP,
    name TEXT
);

CREATE INDEX IF NOT EXISTS idx_sessions_status ON sessions(status);
CREATE INDEX IF NOT EXISTS idx_sessions_updated ON sessions(updated_at);
CREATE INDEX IF NOT EXISTS idx_sessions_name ON sessions(name);
```

### 4.2 JSON Lines 文件 (storage/session/file_storage.py)

对话历史存储在 JSON Lines 文件中：

```
~/.ai/sessions/{session_id}/conversation.jsonl
```

格式示例：
```jsonl
{"id": "msg_xxx", "role": "user", "content": "...", "timestamp": "2026-03-29T10:00:00", "tokens": 150}
{"id": "msg_yyy", "role": "assistant", "content": "...", "timestamp": "2026-03-29T10:00:05", "tokens": 300}
```

```python
class ContextCompactor:
    """上下文压缩器，管理会话消息的压缩"""

    DEFAULT_CONTEXT_LIMIT = 100000   # ~100k tokens
    COMPACTION_THRESHOLD = 0.95      # 95% 时触发

    def __init__(self, context_limit: int = DEFAULT_CONTEXT_LIMIT) -> None:
        """初始化压缩器"""

        """检查是否需要压缩"""
```

## 7. 上下文压缩机制

### 7.1 触发条件

当会话消息总 token 数达到 context limit 的 95% 时触发压缩：

```python
threshold = context_limit * 0.95  # 95000 tokens for 100k limit
should_compact = total_tokens >= threshold
```

### 7.2 压缩算法

1. **保留最近消息**：始终保留最后 10 条消息
2. **分组摘要**：将早期消息按角色分组
3. **生成摘要**：创建压缩摘要消息替换原始消息

```python
async def compact(self, messages: list[SessionMessage]) -> CompactionResult:
    # 分离最近消息和旧消息
    recent = messages[-PRESERVE_RECENT_COUNT:]
    old = messages[:-PRESERVE_RECENT_COUNT]

    # 摘要旧消息
    summarized = await self._summarize_batch(old)

    # 返回压缩结果
    return CompactionResult(
        original_count=len(messages),
        compacted_count=len(summarized) + len(recent),
        original_tokens=total_tokens,
        compacted_tokens=estimated,
        messages=summarized + recent,
    )
```

## 8. 使用示例

### 8.1 创建和管理会话

```python
from mozi.orchestrator.session.manager import SessionManager
from mozi.orchestrator.session.models import SessionMessage, SessionState
from mozi.storage.session.file_storage import FileSessionStorage
from mozi.orchestrator.session.compactor import ContextCompactor

# 初始化
storage = FileSessionStorage("~/.ai/sessions")
compactor = ContextCompactor()
manager = SessionManager(storage=storage, compactor=compactor)

# 创建会话
session = await manager.create_session(
    complexity_score=55,
    metadata={"project": "my-project"}
)

# 添加消息
msg = SessionMessage(
    id="msg_001",
    role="user",
    content="Hello, help me with...",
    timestamp=datetime.now(),
    tokens=50,
)
session = await manager.add_message(session.session_id, msg)

# 暂停会话
session = await manager.pause_session(session.session_id)

# 恢复会话
session = await manager.resume_session(session.session_id)

# 完成会话
session = await manager.complete_session(session.session_id)
```

### 8.2 通过名称恢复会话

```python
# CLI: mozi -r my-session "继续开发"
session = await store.get_by_name("my-session")
if not session:
    session = await store.get(name_or_id)  # 回退到 ID 查找
```

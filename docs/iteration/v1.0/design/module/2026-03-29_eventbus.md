# EventBus 模块设计文档

## 1. 模块概述

### 1.1 模块名称

EventBus（事件总线）

### 1.2 职责

EventBus 模块是 Mozi 系统的基础设施层组件，负责系统中各模块之间的异步通信，
通过发布-订阅模式实现解耦的事件传播，支持事件路由、插件通信和异步处理。

### 1.3 核心能力

| 能力             | 说明                                                   |
| ---------------- | ------------------------------------------------------ |
| 异步发布/订阅     | 支持事件的异步发布和订阅，无需同步调用                 |
| 事件路由         | 基于主题/类型的事件路由到对应的订阅者                  |
| 插件通信         | 插件之间通过 EventBus 进行解耦通信                     |
| 订阅管理         | 支持动态订阅/取消订阅，订阅生命周期管理                |
| 事件过滤         | 支持基于条件的事件过滤和模式匹配                       |
| 死信处理         | 处理失败事件的捕获和死信队列                           |
| 事件追溯         | 支持事件历史的记录和回放                               |

### 1.4 核心问题

| 问题 | 说明 |
| ---- | ---- |
| 模块间耦合 | 直接调用导致模块间紧耦合，难以独立演进 |
| 异步通信复杂 | 异步事件的发送、接收、确认机制复杂 |
| 事件路由 | 如何将事件准确路由到订阅者 |
| 事件可靠性 | 事件发布后如何确保被订阅者正确处理 |

### 1.5 难点

| 难点 | 说明 |
| ---- | ---- |
| 通配符订阅 | 支持 `user.*` 这样的通配符订阅模式 |
| 死信处理 | 订阅者处理失败时如何妥善处理，避免丢失 |
| 事件顺序 | 异步环境下如何保证事件处理顺序 |
| 性能开销 | 发布订阅模式带来的内存和计算开销 |

### 1.6 解决方案

| 方案 | 说明 |
| ---- | ---- |
| 发布-订阅模式 | 解耦发布者和订阅者，通过主题匹配 |
| TopicMatcher | 支持 `*` 和 `>` 通配符的 topic 匹配 |
| 死信队列 | 处理失败的事件进入 DLQ，等待人工处理 |
| 异步执行器 | AsyncEventExecutor 异步处理事件，不阻塞主流程 |

## 3. 发布订阅模型

### 3.1 核心模型定义

```python
from dataclasses import dataclass, field
from datetime import datetime
from enum import Enum
from typing import Any, Awaitable, Callable, Optional
from uuid import UUID, uuid4


class EventPriority(Enum):
    """事件优先级"""
    LOW = "low"        # 低优先级
    NORMAL = "normal"  # 普通优先级
    HIGH = "high"      # 高优先级
    CRITICAL = "critical"  # 关键优先级


class DeliveryMode(Enum):
    """投递模式"""
    FIRE_AND_FORGET = "fire_and_forget"  # 发后即忘
    AT_LEAST_ONCE = "at_least_once"       # 至少一次
    EXACTLY_ONCE = "exactly_once"         # 精确一次（不支持跨进程）


@dataclass
class Event:
    """事件结构"""
    id: str = field(default_factory=lambda: str(uuid4()))
    topic: str = ""                        # 事件主题
    event_type: str = ""                   # 事件类型
    payload: dict[str, Any] = field(default_factory=dict)  # 事件载荷
    timestamp: datetime = field(default_factory=datetime.now)
    priority: EventPriority = EventPriority.NORMAL
    delivery_mode: DeliveryMode = DeliveryMode.FIRE_AND_FORGET
    correlation_id: Optional[str] = None  # 关联 ID，用于事件链追踪
    source: Optional[str] = None          # 事件来源模块
    metadata: dict[str, Any] = field(default_factory=dict)


@dataclass
class Subscription:
    """订阅结构"""
    id: str = field(default_factory=lambda: str(uuid4()))
    topic_pattern: str = ""               # 主题模式（支持通配符）
    event_types: list[str] = field(default_factory=list)  # 感兴趣的事件类型
    callback: Callable[[Event], Awaitable[None]] = field(default=None)
    priority: int = 0                      # 订阅优先级（数值越大越先执行）
    filter_fn: Optional[Callable[[Event], bool]] = None  # 额外过滤函数
    metadata: dict[str, Any] = field(default_factory=dict)
```

## 4. 事件路由

### 4.1 路由器实现

```python
from dataclasses import dataclass
from typing import Awaitable, Callable, Optional
import re


class TopicMatcher:
    """主题匹配器"""

    def __init__(self) -> None:
        self._compiled_patterns: dict[str, re.Pattern] = {}

    def compile_pattern(self, pattern: str) -> re.Pattern:
        """编译主题模式为正则表达式"""
        if pattern in self._compiled_patterns:
            return self._compiled_patterns[pattern]

        # 将 MQTT 风格通配符转换为正则
        # * 匹配单层
        regex_pattern = pattern.replace(".", "\\.")
        regex_pattern = regex_pattern.replace("*", "[^.]+")
        regex_pattern = regex_pattern.replace("#", ".+")

        compiled = re.compile(f"^{regex_pattern}$")
        self._compiled_patterns[pattern] = compiled
        return compiled

    def match(self, pattern: str, topic: str) -> bool:
        """检查主题是否匹配模式"""
        compiled = self.compile_pattern(pattern)
        return compiled.match(topic) is not None


class EventRouter:
    """事件路由器"""

    def __init__(self, matcher: Optional[TopicMatcher] = None) -> None:
        self._matcher = matcher or TopicMatcher()

    def route(
        self,
        event: Event,
        subscriptions: list[Subscription],
    ) -> list[Subscription]:
        """
        将事件路由到匹配的订阅者

        Args:
            event: 待路由的事件
            subscriptions: 所有活跃订阅

        Returns:
            匹配该事件的订阅列表（按优先级排序）
        """
        matched: list[tuple[int, Subscription]] = []

        for sub in subscriptions:
            if self._matches_subscription(event, sub):
                matched.append((sub.priority, sub))

        # 按优先级降序排序
        matched.sort(key=lambda x: x[0], reverse=True)
        return [sub for _, sub in matched]

    def _matches_subscription(self, event: Event, sub: Subscription) -> bool:
        """检查事件是否匹配订阅"""
        # 检查主题匹配
        if not self._matcher.match(sub.topic_pattern, event.topic):
            return False

        # 检查事件类型
        if sub.event_types and event.event_type not in sub.event_types:
            return False

        # 检查过滤函数
        if sub.filter_fn and not sub.filter_fn(event):
            return False

        return True
```

### 4.2 路由决策流程

```
EventBus.publish(topic, event_type, payload)
    │
    ▼
EventRouter.route()
    │
    ├──► 匹配主题模式（TopicMatcher）
    │         │
    │         ▼
    │    匹配成功 ──► 检查事件类型过滤
    │         │
    │         ▼
    │    匹配成功 ──► 检查 filter_fn
    │         │
    │         ▼
    │    filter_fn 返回 True ──► 添加到路由结果
    │
    └──► 返回匹配的订阅列表（按优先级排序）
```

## 5. 异步处理

### 5.1 异步执行器

```python
from concurrent.futures import ThreadPoolExecutor, Future
from typing import Awaitable, Callable, Optional
import asyncio


@dataclass
class ExecutionResult:
    """执行结果"""
    subscription_id: str
    success: bool
    result: Optional[Any] = None
    error: Optional[Exception] = None
    execution_time_ms: float = 0.0


class AsyncEventExecutor:
    """异步事件执行器"""

    def __init__(
        self,
        max_workers: int = 10,
        max_queue_size: int = 1000,
        default_timeout_seconds: float = 30.0,
    ) -> None:
        self._max_workers = max_workers
        self._max_queue_size = max_queue_size
        self._default_timeout = default_timeout_seconds
        self._executor = ThreadPoolExecutor(max_workers=max_workers)
        self._dead_letter_queue: asyncio.Queue[Event] = asyncio.Queue(
            maxsize=max_queue_size
        )

    async def execute(
        self,
        event: Event,
        subscription: Subscription,
        timeout: Optional[float] = None,
    ) -> ExecutionResult:
        """
        异步执行订阅回调

        Args:
            event: 事件对象
            subscription: 匹配的订阅
            timeout: 超时时间（秒）

        Returns:
            执行结果
        """
        start_time = asyncio.get_event_loop().time()
        callback = subscription.callback

        try:
            # 使用 asyncio.wait_for 设置超时
            result = await asyncio.wait_for(
                callback(event),
                timeout=timeout or self._default_timeout,
            )
            execution_time = (
                asyncio.get_event_loop().time() - start_time
            ) * 1000

            return ExecutionResult(
                subscription_id=subscription.id,
                success=True,
                result=result,
                execution_time_ms=execution_time,
            )

        except asyncio.TimeoutError:
            execution_time = (
                asyncio.get_event_loop().time() - start_time
            ) * 1000
            return ExecutionResult(
                subscription_id=subscription.id,
                success=False,
                error=TimeoutError(
                    f"Subscription {subscription.id} timed out after "
                    f"{timeout or self._default_timeout}s"
                ),
                execution_time_ms=execution_time,
            )

        except Exception as e:
            execution_time = (
                asyncio.get_event_loop().time() - start_time
            ) * 1000
            return ExecutionResult(
                subscription_id=subscription.id,
                success=False,
                error=e,
                execution_time_ms=execution_time,
            )

    async def execute_all(
        self,
        event: Event,
        subscriptions: list[Subscription],
    ) -> list[ExecutionResult]:
        """
        并行执行所有匹配的订阅回调

        Args:
            event: 事件对象
            subscriptions: 匹配的订阅列表

        Returns:
            所有执行结果列表
        """
        tasks = [
            self.execute(event, sub)
            for sub in subscriptions
        ]
        results = await asyncio.gather(*tasks, return_exceptions=False)
        return list(results)

    async def add_to_dead_letter_queue(self, event: Event) -> None:
        """将失败事件添加到死信队列"""
        try:
            self._dead_letter_queue.put_nowait(event)
        except asyncio.QueueFull:
            # 死信队列满，丢弃最旧的事件
            try:
                self._dead_letter_queue.get_nowait()
            except asyncio.QueueEmpty:
                pass
            self._dead_letter_queue.put_nowait(event)

    async def get_dead_letter_events(
        self,
        max_count: int = 100,
    ) -> list[Event]:
        """获取死信事件（用于重试或人工处理）"""
        events = []
        for _ in range(max_count):
            try:
                event = self._dead_letter_queue.get_nowait()
                events.append(event)
            except asyncio.QueueEmpty:
                break
        return events
```

### 5.2 并发控制策略

| 策略          | 说明                                           | 适用场景                 |
| ------------- | ---------------------------------------------- | ------------------------ |
| 串行执行      | 按优先级顺序逐个执行                           | 事件有严格顺序要求       |
| 并行执行      | 所有订阅回调并行执行                           | 订阅之间无依赖           |
| 级别隔离      | 高优先级并行，低优先级串行                     | 需要保障关键事件处理     |
| 令牌桶限流    | 限制每秒处理事件数量                           | 防止过载                 |

## 6. 插件通信

### 6.1 插件间通信模型

```python
class PluginCommunication:
    """插件间通信接口"""

    def __init__(self, eventbus: "EventBus") -> None:
        self._eventbus = eventbus
        self._plugin_id: Optional[str] = None

    def register_plugin(self, plugin_id: str, plugin_name: str) -> None:
        """注册插件到 EventBus"""
        self._plugin_id = plugin_id
        self._eventbus.subscribe(
            topic_pattern="plugin.communication",
            event_types=["plugin_message"],
            callback=self._handle_plugin_message,
            metadata={"plugin_id": plugin_id, "plugin_name": plugin_name},
        )

    async def send_to_plugin(
        self,
        target_plugin_id: str,
        payload: dict[str, Any],
        correlation_id: Optional[str] = None,
    ) -> None:
        """向指定插件发送消息"""
        if self._plugin_id is None:
            raise PluginNotRegisteredError()

        event = Event(
            topic="plugin.communication",
            event_type="plugin_message",
            payload={
                "target_plugin_id": target_plugin_id,
                "source_plugin_id": self._plugin_id,
                "payload": payload,
            },
            correlation_id=correlation_id,
        )
        await self._eventbus.publish(event)

    async def broadcast(
        self,
        payload: dict[str, Any],
        exclude_plugins: Optional[list[str]] = None,
    ) -> None:
        """广播消息到所有插件"""
        if self._plugin_id is None:
            raise PluginNotRegisteredError()

        event = Event(
            topic="plugin.communication",
            event_type="plugin_broadcast",
            payload={
                "source_plugin_id": self._plugin_id,
                "payload": payload,
                "exclude_plugins": exclude_plugins or [],
            },
        )
        await self._eventbus.publish(event)

    async def _handle_plugin_message(self, event: Event) -> None:
        """处理收到的插件消息（由子类实现）"""
        raise NotImplementedError("Subclass must implement this method")
```

### 6.2 插件通信协议

| 消息类型        | 方向              | Payload                            | 说明             |
| --------------- | ----------------- | ---------------------------------- | ---------------- |
| `plugin_message` | 点对点           | `{target, source, payload}`        | 发送给特定插件   |
| `plugin_broadcast` | 广播            | `{source, payload, exclude}`       | 广播给所有插件   |
| `plugin_response` | 响应            | `{correlation_id, payload}`        | 响应之前的消息   |

## 7. 接口设计

### 7.1 核心类定义

```python
from abc import ABC, abstractmethod
from typing import Any, Awaitable, Callable, Optional


class BaseEventBus(ABC):
    """EventBus 抽象基类"""

    @abstractmethod
    async def publish(self, event: Event) -> None:
        """发布事件"""
        ...

    @abstractmethod
    async def subscribe(
        self,
        topic_pattern: str,
        event_types: Optional[list[str]] = None,
        callback: Callable[[Event], Awaitable[None]] = None,
        priority: int = 0,
        filter_fn: Optional[Callable[[Event], bool]] = None,
        metadata: Optional[dict[str, Any]] = None,
    ) -> Subscription:
        """订阅事件"""
        ...

    @abstractmethod
    async def unsubscribe(self, subscription_id: str) -> bool:
        """取消订阅"""
        ...

    @abstractmethod
    async def get_subscriptions(
        self,
        topic_pattern: Optional[str] = None,
    ) -> list[Subscription]:
        """获取活跃订阅列表"""
        ...


class EventBus(BaseEventBus):
    """EventBus 实现"""

    def __init__(
        self,
        router: Optional[EventRouter] = None,
        executor: Optional[AsyncEventExecutor] = None,
        enable_dead_letter: bool = True,
        enable_event_history: bool = True,
        max_history_size: int = 1000,
    ) -> None:
        self._router = router or EventRouter()
        self._executor = executor or AsyncEventExecutor()
        self._enable_dead_letter = enable_dead_letter
        self._enable_event_history = enable_event_history
        self._max_history_size = max_history_size

        self._subscriptions: dict[str, Subscription] = {}
        self._event_history: list[Event] = []
        self._lock = asyncio.Lock()

    async def publish(self, event: Event) -> None:
        """发布事件到所有匹配的订阅者"""
        async with self._lock:
            # 记录事件历史
            if self._enable_event_history:
                self._event_history.append(event)
                if len(self._event_history) > self._max_history_size:
                    self._event_history.pop(0)

        # 路由事件到匹配的订阅者
        matched_subscriptions = self._router.route(
            event,
            list(self._subscriptions.values()),
        )

        if not matched_subscriptions:
            return

        # 异步执行所有匹配的订阅回调
        results = await self._executor.execute_all(
            event,
            matched_subscriptions,
        )

        # 处理失败事件
        if self._enable_dead_letter:
            failed_results = [
                r for r in results if not r.success
            ]
            if failed_results:
                await self._handle_failed_events(event, failed_results)

    async def subscribe(
        self,
        topic_pattern: str,
        event_types: Optional[list[str]] = None,
        callback: Callable[[Event], Awaitable[None]] = None,
        priority: int = 0,
        filter_fn: Optional[Callable[[Event], bool]] = None,
        metadata: Optional[dict[str, Any]] = None,
    ) -> Subscription:
        """订阅事件"""
        subscription = Subscription(
            topic_pattern=topic_pattern,
            event_types=event_types or [],
            callback=callback,
            priority=priority,
            filter_fn=filter_fn,
            metadata=metadata or {},
        )

        async with self._lock:
            self._subscriptions[subscription.id] = subscription

        return subscription

    async def unsubscribe(self, subscription_id: str) -> bool:
        """取消订阅"""
        async with self._lock:
            if subscription_id in self._subscriptions:
                del self._subscriptions[subscription_id]
                return True
            return False

    async def get_subscriptions(
        self,
        topic_pattern: Optional[str] = None,
    ) -> list[Subscription]:
        """获取活跃订阅列表"""
        subscriptions = list(self._subscriptions.values())
        if topic_pattern:
            subscriptions = [
                s for s in subscriptions
                if self._router._matcher.match(topic_pattern, s.topic_pattern)
            ]
        return subscriptions

    async def get_event_history(
        self,
        topic: Optional[str] = None,
        event_type: Optional[str] = None,
        limit: int = 100,
    ) -> list[Event]:
        """获取事件历史"""
        events = list(self._event_history)
        if topic:
            events = [e for e in events if e.topic == topic]
        if event_type:
            events = [e for e in events if e.event_type == event_type]
        return events[-limit:]

    async def _handle_failed_events(
        self,
        event: Event,
        failed_results: list[ExecutionResult],
    ) -> None:
        """处理失败的事件"""
        # 记录错误日志
        for result in failed_results:
            error_info = {
                "subscription_id": result.subscription_id,
                "error": str(result.error),
                "execution_time_ms": result.execution_time_ms,
            }
            event.metadata["_failed_deliveries"] = (
                event.metadata.get("_failed_deliveries", []) + [error_info]
            )

        # 添加到死信队列
        await self._executor.add_to_dead_letter_queue(event)
```

### 7.2 便捷发布方法

```python
class EventBusHelpers:
```

## 8. 数据流

### 8.1 与 Orchestrator 的交互

```
Orchestrator
    │
    ├──► EventBus.publish("agent.message", "user_message")
    │         │
    │         ▼
    │    EventRouter.route() ──► 匹配所有订阅
    │         │
    │         ▼
    │    AsyncEventExecutor.execute_all()
    │         │
    │         ▼
    │    各订阅者处理（Model、Tools 等）
    │
    ├──► EventBus.subscribe("agent.message", callback)
    │         │
    │         ▼
    │    SessionManager 更新会话
    │
    └──► EventBus.publish("orchestrator.complexity", "complexity_evaluated")
              │
              ▼
         各模块订阅结果并响应
```

### 8.2 与 Tools 的交互

```
Tools 模块
    │
    ├──► EventBus.subscribe("agent.message", event_type="tool_result")
    │         │
    │         ▼
    │    接收工具执行结果 ──► 处理结果 ──► 返回给 Orchestrator
    │
    ├──► Tools.execute() ─► 执行工具
    │
    └──► EventBus.publish("agent.message", "tool_result")
              │
              ▼
         EventBus ──► Orchestrator 接收
```

### 8.3 与 Memory 的交互

```
Memory 模块
    │
    ├──► EventBus.subscribe("agent.message", callback)
    │         │
    │         ▼
    │    监听 user_message ─► 提取关键信息
    │
    ├──► EventBus.publish("memory.persist", "memory_stored")
    │         │
    │         ▼
    │    Storage 模块持久化
    │
    └──► Memory.recall() ─► 召回长期记忆
              │
              ▼
         注入到 Context
```

### 8.4 与插件系统的交互

```
Plugin-A                               Plugin-B
    │                                      │
    │  PluginCommunication.send_to_plugin() │
    │ ─────────────────────────────────────►│
    │     EventBus.publish("plugin.communication")
    │              │
    │              ▼
    │         Plugin-B 的订阅回调被触发
    │              │
    │              ▼
    │         处理来自 Plugin-A 的消息
```

### 8.5 EventBus 内部数据流

```
发布事件
    │
    ▼
Event ──► EventRouter.route()
    │         │
    │         ▼
    │    匹配订阅者（按主题 + 类型 + filter）
    │         │
    │         ▼
    │    按优先级排序
    │         │
    │         ▼
    └──► AsyncEventExecutor.execute_all()
              │
              ├──► 订阅者-1 回调 ──► ExecutionResult
              ├──► 订阅者-2 回调 ──► ExecutionResult
              └──► ...
                   │
                   ▼
              失败事件 ──► DeadLetterQueue
                   │
                   ▼
              重试或人工处理
```

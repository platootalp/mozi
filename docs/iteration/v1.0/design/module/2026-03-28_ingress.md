## 1. 模块概述

Ingress 模块是系统的入口层，负责接收用户输入并返回系统响应。支持三种接入方式：CLI 命令行、Web UI、IDE 插件。

### 1.1 核心职责

- 命令解析与参数验证
- 用户会话管理
- 响应渲染与格式化
- 认证与授权（V2.0）

### 1.2 差异化特性

| 特性 | 说明 |
|------|------|
| 多端一致 | CLI/Web/IDE 体验一致 |
| 流式输出 | 支持 Server-Sent Events |
| 命令补全 | 支持 Tab 自动补全 |
| 多会话管理 | 支持并发会话 |

## 4. 核心工作流

### 4.1 CLI 交互流程

```
用户输入
    │
    ▼
┌─────────────────┐
│  Typer CLI     │ ← 命令解析 & 参数验证
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  SessionManager │ ← 会话创建/恢复
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Orchestrator   │ → 转发到编排层
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  Renderer       │ ← 响应格式化
└────────┬────────┘
         │
         ▼
终端输出 / JSON
```

### 4.2 WebSocket 实时通信流程

```
客户端
    │
    ├──→ WS Connect ──→ SessionManager.register()
    │
    ├──→ Send Message ──→ Orchestrator.execute()
    │                        │
    │                        ▼
    │                    Agent 处理中...
    │                        │
    │←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←┘
    │
    └──→ Disconnect ──→ SessionManager.unregister()
```

## 7. Claude Code CLI Commands

### 7.1 Claude Code CLI Interface

Claude Code provides a comprehensive CLI interface with the following patterns:

| Command/Option | 说明 |
|----------------|------|
| `claude` | Start interactive session |
| `claude [command]` | Execute specific command |
| `claude -p PROMPT` | Direct prompt execution (non-interactive) |
| `claude --dangerously-skip-permissions` | Skip all permission prompts |
| `claude --output-format=json` | JSON output |
| `claude --print` | Print result only (no spinner) |
| `claude --verbose` | Verbose output |
| `claude --resume` | Resume a session |
| `claude --model <model>` | Specify model |
| `claude --max-tokens <limit>` | Token limit |

### 7.2 Mozi vs Claude Code CLI Comparison

| Feature | Claude Code | Mozi (Current) | Gap |
|---------|-------------|----------------|-----|
| Interactive session | `claude` | `mozi` | Equivalent |
| Direct prompt | `-p PROMPT` | `-p "query"` | Equivalent |
| Skip permissions | `--dangerously-skip-permissions` | N/A | Missing |
| JSON output | `--output-format=json` | N/A | Missing |
| Print only (no spinner) | `--print` | N/A | Missing |
| Verbose mode | `--verbose` | N/A | Missing |
| Resume session | `--resume` | `-r <session>` | Equivalent |
| Specify model | `--model` | `--model` | Equivalent |
| Token limit | `--max-tokens` | N/A | Missing |
| Config file | N/A | `--config <path>` | Mozi only |
| Session list | N/A | `list` subcommand | Mozi only |
| Session delete | N/A | `delete <session>` | Mozi only |

### 7.3 CLI Patterns Mozi Should Adopt

1. **Permission Skip Option**: Add `--dangerously-skip-permissions` for CI/automated environments
   - Use case: Batch processing, testing, scripted workflows
   - Security consideration: Should require confirmation or env var override

2. **Output Format Control**: Add `--output-format=json/text`
   - Enables programmatic parsing
   - Consistent with `--print` for clean output

3. **No-Spinner Mode**: Add `--print` for quiet output
   - Suitable for log files and redirected output
   - Better for integration with other tools

4. **Verbose Mode**: Add `--verbose` for detailed logging
   - Debug mode for troubleshooting
   - Could output internal reasoning steps

5. **Token Limit Control**: Add `--max-tokens` parameter
   - Control response length
   - Cost management for large tasks

### 7.4 .claudeignore Support

Claude Code uses `.claudeignore` to exclude files from context. Mozi should adopt similar functionality:

**Implementation recommendations:**

| Pattern | Behavior |
|---------|----------|
| `# comment` | Lines starting with # are comments |
| `path/to/dir/` | Trailing slash denotes directory |
| `*.log` | Glob patterns for file types |
| `path/to/file` | Specific files |
| `!important/*` | Negation with ! to re-include |

**Use cases:**
- Exclude dependency directories (`node_modules/`, `venv/`, `.venv/`)
- Exclude build artifacts (`dist/`, `build/`, `*.pyc`)
- Exclude sensitive files (`.env`, `*.key`)
- Exclude documentation files when not relevant to task

**File location:**
- Project-level: `.claudeignore` in project root
- User-level: Respect global gitignore settings

# Task 模块设计文档

## 1. 模块概述

### 1.1 模块名称

Task（任务规划）

### 1.2 职责

Task 模块是 Mozi AI Coding Agent 的核心层组件之一，负责复杂任务的规划与执行管理。它承担以下核心职责：

- **任务分解**：将复杂任务拆分为可执行的子任务
- **依赖分析**：分析子任务间的依赖关系，构建执行计划
- **进度跟踪**：监控任务执行状态，维护执行进度
- **失败恢复**：检测执行失败，执行回滚或重试策略
- **结果汇总**：整合多子任务执行结果，生成最终输出

### 1.3 核心能力

| 能力 | 说明 |
| --- | --- |
| 任务建模 | 支持任务的定义、状态管理、优先级排序 |
| 智能分解 | 基于 LLM 的任务分解，支持串行/并行执行计划 |
| 依赖管理 | 自动分析任务依赖，生成拓扑排序执行序列 |
| 状态跟踪 | 实时监控任务状态，支持暂停/恢复/取消操作 |
| 失败处理 | 多级重试策略、自动回滚机制、优雅降级 |
| 结果聚合 | 汇总多个子任务结果，支持部分成功场景 |

### 1.4 核心问题

| 问题 | 说明 |
| --- | --- |
| 任务分解 | 如何将复杂任务拆分为可执行的子任务 |
| 依赖管理 | 子任务间的依赖关系如何管理 |
| 执行进度 | 如何跟踪任务执行进度 |
| 失败恢复 | 任务失败后如何恢复或降级 |

### 1.5 难点

| 难点 | 说明 |
| --- | --- |
| 分解质量 | LLM 分解任务的质量和一致性 |
| 循环依赖 | 如何检测和避免任务间的循环依赖 |
| 部分成功 | 多子任务场景下如何处理部分失败 |
| 状态一致性 | 并行执行时的状态同步问题 |

### 1.6 解决方案

| 方案 | 说明 |
| --- | --- |
| TaskDecomposer | 基于 LLM 的智能任务分解 |
| DependencyAnalyzer | 拓扑排序 + 循环检测 |
| ProgressTracker | 实时状态跟踪，支持暂停/恢复 |
| RecoveryManager | 重试、回滚、降级多级策略 |

### 1.7 在架构中的位置

Task 模块位于核心层（Core），在 COMPLEX 复杂度任务路由时由 Orchestrator 调用：

```
Orchestrator（COMPLEX 路由）
    │
    ▼
Planner 分解任务
    │
    ▼
Task 模块管理执行
    │
    ├──► Agent-A 执行子任务-1
    ├──► Agent-B 执行子任务-2（可并行）
    └──► Agent-C 执行子任务-3
              │
              ▼
        Task 模块汇总结果
              │
              ▼
        返回 Orchestrator
```

```python
    子任务数据模型

    description: str = ""
    status: TaskStatus = TaskStatus.PENDING

    assigned_agent: str | None = None  # 分配的 Agent 类型

    retry_count: int = 0
    max_retries: int = 3
    execution_order: int = 0           # 执行顺序（拓扑排序后）
    metadata: dict[str, Any] = field(default_factory=dict)

    def can_execute(self, completed_tasks: set[UUID]) -> bool:

        判断子任务是否可以执行

        Args:
            completed_tasks: 已完成的子任务 ID 集合

        Returns:

        return all(dep_id in completed_tasks for dep_id in self.dependencies)
```

## 4. 任务分解

### 4.1 分解策略

任务分解采用 **LLM 驱动 + 规则约束** 的混合策略：

1. **LLM 分解层**：使用模型理解任务意图，生成候选子任务列表
2. **规则验证层**：验证分解结果的完整性和可执行性
3. **优化层**：合并过细粒度任务，拆分过粗粒度任务

### 4.2 分解算法

```python
class TaskDecomposer:
    """任务分解器"""

    def __init__(
        self,
        model_adapter: ModelAdapter,
        min_subtasks: int = 1,
        max_subtasks: int = 10,
    ) -> None:
        self._model = model_adapter
        self._min_subtasks = min_subtasks
        self._max_subtasks = max_subtasks

    async def decompose(
        self,
        task: Task,
        context: dict[str, Any],
    ) -> list[SubTask]:
        """
        分解任务为子任务

        Args:
            task: 待分解的任务
            context: 分解上下文（项目结构、相关文件、Memory 数据等）

        Returns:
            list[SubTask]: 分解后的子任务列表
        """
        # 1. 构建分解 prompt
        prompt = self._build_decompose_prompt(task, context)

        # 2. 调用 LLM 生成子任务
        response = await self._model.invoke(
            messages=[{"role": "user", "content": prompt}]
        )

        # 3. 解析 LLM 输出
        subtasks = self._parse_subtasks(response.content)

        # 4. 验证和优化
        subtasks = self._validate_and_optimize(subtasks)

        # 5. 设置执行顺序
        subtasks = self._assign_execution_order(subtasks)

        return subtasks

    def _build_decompose_prompt(
        self,
        task: Task,
        context: dict[str, Any],
    ) -> str:
        """构建分解提示词"""
        return f"""将以下任务分解为可执行的子任务。

任务：{task.title}
描述：{task.description}

项目上下文：
- 目录结构：{context.get('dir_structure', 'N/A')}
- 相关文件：{context.get('related_files', [])}
- 技术栈：{context.get('tech_stack', 'N/A')}

要求：
1. 每个子任务应可独立执行或与依赖任务并行执行
2. 子任务数量控制在 {self._min_subtasks} 到 {self._max_subtasks} 之间
3. 每个子任务应包含清晰的输入输出定义
4. 标注子任务间的依赖关系

请以 JSON 格式输出子任务列表。"""
```

## 5. 依赖分析

### 5.1 依赖类型

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| SEQUENTIAL | 串行依赖，必须等待前一个完成 | 单元测试依赖代码编写 |
| PARALLEL | 可并行，无依赖关系 | 多个独立功能的实现 |
| CONDITIONAL | 条件依赖，满足条件时触发 | 失败重试后的分支 |

### 5.2 依赖分析器

```python
class DependencyAnalyzer:
    """依赖分析器"""

    def __init__(self) -> None:
        self._cache: dict[UUID, list[UUID]] = {}

    def analyze(
        self,
        subtasks: list[SubTask],
    ) -> DependencyGraph:
        """
        分析子任务间的依赖关系

        Args:
            subtasks: 子任务列表

        Returns:
            DependencyGraph: 依赖图
        """
        # 1. 构建初始依赖关系
        dependencies = self._build_initial_dependencies(subtasks)

        # 2. 检测隐式依赖（共享资源、文件冲突等）
        implicit_deps = self._detect_implicit_dependencies(subtasks)

        # 3. 合并依赖关系
        all_dependencies = self._merge_dependencies(dependencies, implicit_deps)

        # 4. 检测循环依赖
        cycles = self._detect_cycles(all_dependencies)
        if cycles:
            raise CyclicDependencyError(f"Cyclic dependencies detected: {cycles}")

        # 5. 生成依赖图
        return DependencyGraph(all_dependencies)

    def generate_execution_plan(
        self,
        graph: DependencyGraph,
    ) -> list[list[SubTask]]:
        """

            graph: 依赖图

        Returns:
            list[list[SubTask]]: 分层执行计划，每层内的任务可并行执行
```

```python
    """依赖图"""
    nodes: dict[UUID, SubTask] = field(default_factory=dict)
    edges: list[DependencyEdge] = field(default_factory=list)

    def get_executable_tasks(
        self,
        completed: set[UUID],
    ) -> list[UUID]:
        """
        获取当前可执行的任务

        Args:
            completed: 已完成的任务 ID 集合

        Returns:
            list[UUID]: 可执行的任务 ID 列表（按优先级排序）
        """
        executable = []
        for node_id, node in self.nodes.items():
            if node_id in completed:
                continue
            if node.can_execute(completed):
                executable.append(node_id)

        # 按优先级排序
        return sorted(
            executable,
            key=lambda tid: self.nodes[tid].priority.value,
            reverse=True,
        )

    def get_parallel_layers(
        self,
    ) -> list[list[UUID]]:
        """
        获取并行执行层级

        Returns:
            list[list[UUID]]: 每层的任务 ID 列表，层级越高越晚执行
        """
        return topological_sort分层(self)
```

### 5.4 循环依赖检测

```python
def _detect_cycles(
    self,
    dependencies: dict[UUID, list[UUID]],
) -> list[list[UUID]]:
    """使用 DFS 检测循环依赖"""
    visited: set[UUID] = set()
    rec_stack: set[UUID] = set()
    cycles: list[list[UUID]] = []

    def dfs(node: UUID, path: list[UUID]) -> None:
        visited.add(node)
        rec_stack.add(node)
        path.append(node)

        for neighbor in dependencies.get(node, []):
            if neighbor not in visited:
                dfs(neighbor, path.copy())
            elif neighbor in rec_stack:
                # 发现循环
                cycle_start = path.index(neighbor)
                cycles.append(path[cycle_start:] + [neighbor])

        rec_stack.remove(node)

    for node in dependencies:
        if node not in visited:
            dfs(node, [])

    return cycles
```

## 6. 进度跟踪

### 6.1 进度跟踪器

```python
class ProgressTracker:
    """进度跟踪器"""

    def __init__(self, task: Task) -> None:
        self._task = task
        self._start_time: datetime | None = None
        self._completed_subtasks: dict[UUID, SubTaskResult] = {}
        self._failed_subtasks: dict[UUID, SubTaskResult] = {}
        self._running_subtasks: set[UUID] = set()
        self._pause_event = asyncio.Event()
        self._pause_event.set()  # 默认不暂停

    @property
    def progress(self) -> float:
        """
        计算任务进度

        Returns:
            float: 0.0 到 1.0 之间的进度值
        """
        if not self._task.subtasks:
            return 0.0

        total = len(self._task.subtasks)
        completed = len(self._completed_subtasks)
        failed = len(self._failed_subtasks)

        # 部分成功也算完成
        return (completed + failed) / total

    @property
    def status(self) -> TaskStatus:
        """获取当前任务状态"""
        if self._pause_event.is_set() and self._running_subtasks:
            return TaskStatus.PAUSED
        if self._failed_subtasks and not self._running_subtasks:
            if self._can_retry():
                return TaskStatus.RUNNING
            return TaskStatus.FAILED
        if len(self._completed_subtasks) + len(self._failed_subtasks) == len(
            self._task.subtasks
        ):
            return TaskStatus.COMPLETED
        if self._running_subtasks:
            return TaskStatus.RUNNING
        return TaskStatus.PENDING

    def update_subtask_status(
        self,
        subtask_id: UUID,
        status: TaskStatus,
        result: SubTaskResult | None = None,
    ) -> None:
        """
        更新子任务状态

        Args:
            subtask_id: 子任务 ID
            status: 新状态
            result: 执行结果（如果已完成）
        """
        if status == TaskStatus.RUNNING:
            self._running_subtasks.add(subtask_id)
        elif status == TaskStatus.COMPLETED and result:
            self._running_subtasks.discard(subtask_id)
            self._completed_subtasks[subtask_id] = result
        elif status == TaskStatus.FAILED and result:
            self._running_subtasks.discard(subtask_id)
            self._failed_subtasks[subtask_id] = result

    async def wait_if_paused(self) -> None:
        """如果任务暂停则等待"""
        await self._pause_event.wait()

    def pause(self) -> None:
        """暂停任务"""
        self._pause_event.clear()

    def resume(self) -> None:
        """恢复任务"""
        self._pause_event.set()

    def get_snapshot(self) -> TaskSnapshot:
        """获取任务进度快照"""
        return TaskSnapshot(
```

## 7. 失败恢复

### 7.1 恢复策略

| 策略 | 触发条件 | 行为 |
| --- | --- | --- |
| 重试 | 单个子任务失败 | 使用指数退避重试，最多重试 max_retries 次 |
| 跳过 | 子任务失败但非关键 | 标记为失败，继续执行其他子任务 |
| 回滚 | 关键子任务失败 | 回滚所有已完成的子任务 |
| 降级 | 重试耗尽 | 返回部分结果，记录失败项 |
| 放弃 | 不可恢复错误 | 终止任务，记录完整错误 |

### 7.2 重试机制

```python
class RetryStrategy:
    """重试策略"""

    def __init__(
        self,
        max_retries: int = 3,
        base_delay: float = 1.0,
        max_delay: float = 60.0,
        exponential_base: float = 2.0,
    ) -> None:
        self._max_retries = max_retries
        self._base_delay = base_delay
        self._max_delay = max_delay
        self._exponential_base = exponential_base

    def calculate_delay(self, attempt: int) -> float:
        """
        计算重试延迟

        Args:
            attempt: 当前尝试次数（从 1 开始）

        Returns:
            float: 延迟秒数
        """
        delay = self._base_delay * (self._exponential_base ** (attempt - 1))
        return min(delay, self._max_delay)

    def should_retry(self, attempt: int, error: Exception) -> bool:
        """
        判断是否应该重试

        Args:
            attempt: 当前尝试次数
            error: 发生的错误

        Returns:
            bool: 是否应该重试
        """
        if attempt >= self._max_retries:
            return False

        # 可重试的错误类型
        retryable_errors = (
            TimeoutError,
            ConnectionError,
            TemporaryError,
        )
        return isinstance(error, retryable_errors)
```

### 7.3 回滚机制

```python
class RollbackManager:
    """回滚管理器"""

    def __init__(self) -> None:
        self._rollback_handlers: dict[str, Callable] = {}

    def register_rollback(
        self,
        subtask_id: UUID,
        handler: Callable,
    ) -> None:
        """
        注册回滚处理器

        Args:
            subtask_id: 子任务 ID
            handler: 回滚处理函数
        """
        self._rollback_handlers[str(subtask_id)] = handler

    async def rollback(
        self,
        completed_subtasks: list[UUID],
    ) -> RollbackResult:
        """
        执行回滚

        Args:
            completed_subtasks: 已完成的子任务 ID 列表（按执行顺序）

        Returns:
            RollbackResult: 回滚结果
        """
        result = RollbackResult()
        # 逆序回滚
        for subtask_id in reversed(completed_subtasks):
            handler = self._rollback_handlers.get(str(subtask_id))
            if handler:
                try:
                    await handler()
                    result.rolled_back.append(subtask_id)
                except Exception as e:
                    result.failed.append((subtask_id, str(e)))

        return result
```

### 7.4 失败恢复器

```python
class RecoveryManager:
    """失败恢复管理器"""

    def __init__(
        self,
        retry_strategy: RetryStrategy,
        rollback_manager: RollbackManager,
    ) -> None:
        self._retry_strategy = retry_strategy
        self._rollback_manager = rollback_manager
        self._subtask_retries: dict[UUID, int] = {}

    async def handle_subtask_failure(
        self,
        subtask: SubTask,
        error: Exception,
        tracker: ProgressTracker,
    ) -> RecoveryAction:
        """
        处理子任务失败

        Args:
            subtask: 失败的子任务
            error: 错误信息
            tracker: 进度跟踪器

        Returns:
            RecoveryAction: 建议的恢复动作
        """
        current_retries = self._subtask_retries.get(subtask.id, 0)

        # 判断是否可重试
        if self._retry_strategy.should_retry(current_retries, error):
            delay = self._retry_strategy.calculate_delay(current_retries + 1)
            return RecoveryAction(
                action=RecoveryActionType.RETRY,
                delay_seconds=delay,
                message=f"Retrying in {delay}s (attempt {current_retries + 1})",
            )

        # 判断是否需要回滚
        if self._is_critical_task(subtask):
            return RecoveryAction(
                action=RecoveryActionType.ROLLBACK,
                message="Critical task failed, initiating rollback",
            )

        # 降级处理
        return RecoveryAction(
            action=RecoveryActionType.DEGRADE,
            message="Task failed, continuing with partial results",
        )


@dataclass
class RecoveryAction:
    """恢复动作"""
    action: RecoveryActionType
    delay_seconds: float = 0.0
    message: str = ""


class RecoveryActionType(Enum):
    """恢复动作类型"""
    RETRY = "retry"           # 重试
    ROLLBACK = "rollback"     # 回滚
    DEGRADE = "degrade"       # 降级
    ABANDON = "abandon"       # 放弃
```

```python
class TaskManager(ABC):

    @abstractmethod
    async def create_task(
        self,

        description: str,
        priority: TaskPriority = TaskPriority.NORMAL,

        """
        创建新任务

        Args:
            title: 任务标题
            description: 任务描述
            priority: 任务优先级

        Returns:
            Task: 创建的任务
        """

    @abstractmethod
    async def decompose_task(
        self,

        context: dict[str, Any],
    ) -> list[SubTask]:
        """
        分解任务

        Args:
            task_id: 任务 ID
            context: 分解上下文

        Returns:
            list[SubTask]: 子任务列表
        """

    @abstractmethod
    async def execute_task(
        self,

            task_id: 任务 ID
```

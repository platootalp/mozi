# Tools 模块设计文档

## 1. 模块概述

### 1.1 模块名称

Tools（工具系统）

### 1.2 职责

Tools 模块是 Mozi AI Coding Agent 的核心能力层之一，负责：
- 提供内置工具（Read/Write/Edit/Glob/Grep/Bash）
- 管理外部工具注册与调用（MCP 协议）
- 维护工具注册表，实现工具发现与路由
- 执行安全检查与权限控制

### 1.3 核心能力

| 能力 | 说明 |
| ---- | ---- |
| 内置工具 | 提供 6 种基础文件操作工具，覆盖日常编码需求 |
| 工具注册 | 统一的注册/注销/查询接口 |
| 工具执行 | 异步执行工具，支持超时控制 |
| 安全白名单 | 路径访问白名单验证，防止越权操作 |
| MCP 集成 | 支持通过 MCP 协议扩展外部工具 |

### 1.4 核心问题

| 问题 | 说明 |
| ---- | ---- |
| 工具抽象 | 如何统一不同类型工具（内置/MCP）的调用方式 |
| 工具发现 | 如何让 Agent 知道有哪些工具可用 |
| 安全性 | 工具执行可能带来安全风险 |
| 超时控制 | 工具执行时间不可控，可能长时间阻塞 |

### 1.5 难点

| 难点 | 说明 |
| ---- | ---- |
| 参数验证 | 工具参数的类型和格式验证 |
| 结果解析 | 不同工具返回格式不同，如何统一处理 |
| 权限控制 | 工具执行时的权限检查 |
| MCP 集成 | 外部 MCP 工具的协议适配 |

### 1.6 解决方案

| 方案 | 说明 |
| ---- | ---- |
| Tool 抽象基类 | 统一接口定义，Tool.execute() |
| ToolRegistry | 工具注册表，支持动态注册/注销 |
| ToolContext | 传递安全上下文、权限信息 |
| 超时控制 | 异步执行 + 超时取消 |

## 3. 内置工具

### 3.1 ReadFileTool

读取指定路径的文件内容。

**参数定义：**

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| path | string | 是 | 文件路径 |
| limit | integer | 否 | 最大读取行数 |
| offset | integer | 否 | 起始行偏移（0-indexed） |

**返回值：**

```python
ToolResult(
    success=True,
    output="文件内容字符串",
    metadata={
        "path": "/absolute/path",
        "lines": 100,
        "total_lines": 500
    }
)
```

### 3.2 WriteFileTool

向指定路径写入内容。

**参数定义：**

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| path | string | 是 | 文件路径 |
| content | string | 是 | 写入内容 |
| create_parents | boolean | 否 | 是否创建父目录（默认 False） |

### 3.3 EditFileTool

对文件进行查找替换操作。

**参数定义：**

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| path | string | 是 | 文件路径 |
| old_string | string | 是 | 待替换字符串或正则 |
| new_string | string | 是 | 替换后字符串 |
| use_regex | boolean | 否 | 是否使用正则匹配（默认 False） |

### 3.4 BashTool

在子进程中执行 Bash 命令。

**参数定义：**

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| command | string | 是 | 要执行的命令 |
| timeout | integer | 否 | 超时时间（秒，默认 30） |
| working_directory | string | 否 | 工作目录 |

**返回值：**

```python
ToolResult(
    success=True,
    output={
        "stdout": "命令输出",
        "stderr": "错误输出",
        "exit_code": 0
    }
)
```

### 3.5 GrepTool

在文件内容中搜索匹配的模式。

**参数定义：**

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| pattern | string | 是 | 搜索模式 |
| path | string | 否 | 搜索路径（默认 "."） |
| use_regex | boolean | 否 | 是否正则（默认 True） |
| ignore_case | boolean | 否 | 忽略大小写（默认 False） |
| recursive | boolean | 否 | 递归搜索（默认 True） |
| file_pattern | string | 否 | 文件名过滤模式 |

### 3.6 GlobTool

通过 glob 模式查找文件。

**参数定义：**

| 参数 | 类型 | 必填 | 说明 |
| ---- | ---- | ---- | ---- |
| pattern | string | 是 | glob 模式 |
| path | string | 否 | 搜索基础路径（默认 "."） |
| recursive | boolean | 否 | 递归搜索（默认 True） |

## 4. 工具注册机制

### 4.1 ToolRegistry

工具注册表是工具管理的核心组件，提供以下功能：

**注册工具：**

```python
def register(self, tool: Tool) -> None:
    """注册工具实例到注册表"""
```

**注销工具：**

```python
def unregister(self, name: str) -> bool:
    """从注册表移除工具"""
```

**获取工具：**

```python
def get(self, name: str) -> Tool | None:
    """根据名称查询工具"""
```

**列举工具：**

```python
def list_tools(self) -> list[dict[str, Any]]:
    """列出所有已注册工具的元信息"""
```

**执行工具：**

```python
async def execute(
    self,
    name: str,
    context: ToolContext,
    **kwargs: Any
) -> ToolResult:
    """执行指定工具"""
```

### 4.2 工具发现流程

```
1. 启动时调用 register_all() 注册所有内置工具
2. MCP Client 初始化时注册外部工具
3. 工具名称唯一，重复注册抛出 ValueError
```

### 4.3 内置工具批量注册

```python
def register_all(registry: ToolRegistry | None = None) -> ToolRegistry:
    """注册所有内置工具"""
    if registry is None:
        registry = ToolRegistry()

    tools = [
        ReadFileTool(),
        WriteFileTool(),
        EditFileTool(),
        BashTool(),
        GrepTool(),
        GlobTool(),
    ]

    for tool in tools:
        registry.register(tool)

    return registry
```

## 5. MCP 集成

### 5.1 MCP Client 职责

MCP Client 负责：
- 与外部 MCP 服务器建立连接
- 发现外部服务器提供的工具
- 将外部工具注册到 ToolRegistry
- 转发工具调用到外部服务器

### 5.2 MCP 协议交互流程

```
1. MCP Client 启动，连接到配置的 MCP 服务器
2. 列出服务器可用工具列表
3. 将外部工具适配为本地 Tool 接口
4. 注册到 ToolRegistry
5. 执行时通过 MCP 协议调用远程工具
6. 将结果转换为本地图工具结果格式
```

### 5.3 外部工具适配

MCP Client 将外部工具适配为本地 Tool 接口：

```python
class MCPAdapterTool(Tool):
    """MCP 外部工具适配器"""

    name: str
    description: str
    parameters: dict[str, Any]

    def __init__(self, mcp_client: MCPClient, remote_tool: RemoteTool):
        self._client = mcp_client
        self._remote_tool = remote_tool

    async def execute(self, context: ToolContext, **kwargs: Any) -> ToolResult:
        return await self._client.call_tool(
            self._remote_tool.name,
            kwargs
        )
```

## 6. 安全检查

### 6.1 路径白名单验证

所有文件操作工具必须验证访问路径：

```python
def _validate_path(self, context: ToolContext, path: str) -> Path:
    """验证路径是否在允许范围内"""
    resolved = self._resolve_path(context, path)

    # 获取白名单配置
    whitelist = context.variables.get("path_whitelist", ["."])

    # 检查是否在白名单内
    for allowed in whitelist:
        allowed_path = Path(allowed).resolve()
        if resolved.resolve().is_relative_to(allowed_path):
            return resolved

    raise PermissionError(f"Path not in whitelist: {path}")
```

### 6.2 Bash 命令安全限制

BashTool 执行时应用以下限制：

| 限制项 | 说明 |
| ---- | ---- |
| 超时控制 | 单次执行最长 30 秒（可配置） |
| 工作目录 | 必须在项目目录内 |
| 危险命令 | 禁止执行交互式命令 |

### 6.3 权限模型

```
ToolContext 中包含权限信息：
- working_directory: 允许的工作目录范围
- session_id: 会话标识
- variables: 共享变量（含 path_whitelist）
```

## 7. 接口设计

### 7.1 核心类型定义

```python
from __future__ import annotations

from abc import ABC, abstractmethod
from dataclasses import dataclass, field
from typing import Any


@dataclass
class ToolContext:
    """工具执行上下文"""
    working_directory: str = "."
    session_id: str | None = None
    variables: dict[str, Any] = field(default_factory=dict)
    timeout: int = 30


@dataclass
class ToolResult:
    """工具执行结果"""
    success: bool = False
    output: Any = None
    error: str | None = None
    metadata: dict[str, Any] = field(default_factory=dict)

    def __post_init__(self) -> None:
        if self.success and self.error is not None:
            raise ValueError(
                "Cannot have error message when success is True"
            )


class Tool(ABC):
    """工具抽象基类"""
    name: str
    description: str
    parameters: dict[str, Any] = field(default_factory=dict)

    @abstractmethod
    async def execute(
        self,
        context: ToolContext,
        **kwargs: Any
    ) -> ToolResult:
        raise NotImplementedError
```

### 7.2 ToolRegistry 接口

```python
class ToolRegistry:
    """工具注册表"""

    def __init__(self) -> None:
        self._tools: dict[str, Tool] = {}

    def register(self, tool: Tool) -> None:
        if not isinstance(tool, Tool):
            raise TypeError(
                f"Expected Tool instance, got {type(tool).__name__}"
            )
        if tool.name in self._tools:
            raise ValueError(f"Tool already registered: {tool.name}")
        self._tools[tool.name] = tool

    def unregister(self, name: str) -> bool: ...
    def get(self, name: str) -> Tool | None: ...
    def list_tools(self) -> list[dict[str, Any]]: ...

    async def execute(
        self,
        name: str,
        context: ToolContext,
        **kwargs: Any
    ) -> ToolResult: ...

    def __len__(self) -> int: ...
    def __contains__(self, name: str) -> bool: ...
```

### 7.3 工具参数 Schema 示例

```python
parameters: dict[str, Any] = {
    "type": "object",
    "properties": {
        "path": {
            "type": "string",
            "description": "Path to the file to read",
        },
        "limit": {
            "type": "integer",
            "description": "Maximum number of lines to read",
            "default": None,
        },
    },
    "required": ["path"],
}
```

## 8. 数据流

### 8.1 工具调用数据流

```
Model.invoke()
    │
    ▼
Orchestrator 路由到工具调用
    │
    ▼
ToolRegistry.execute(name, context, **kwargs)
    │
    ├──► 安全检查（白名单验证）
    │
    ├──► 工具发现（registry.get(name)）
    │
    ├──► 工具执行（tool.execute(context, **kwargs)）
    │
    ▼
ToolResult 返回
    │
    ▼
EventBus.publish("tool_result", payload)
    │
    ▼
Context 更新 + Session 记录
```

### 8.2 上下文交互

| 层级 | 与 Tools 的交互 |
| ---- | --------------- |
| Orchestrator | 传递 ToolContext，决定调用哪些工具 |
| Model | 接收工具结果，生成后续响应 |
| Context | 提供 variables（含白名单配置） |
| Session | 记录工具调用历史 |
| EventBus | 发布工具执行事件 |

```python
import pytest
from mozi.capabilities.tools.framework import ToolContext, ToolResult
from mozi.capabilities.tools.builtin import ReadFileTool


@pytest.mark.unit
class TestReadFileTool:
    async def test_read_file_success(self, tmp_path: Path):
        """测试成功读取文件"""
        test_file = tmp_path / "test.txt"
        test_file.write_text("Hello, World!")

        tool = ReadFileTool()
        context = ToolContext(working_directory=str(tmp_path))

        result = await tool.execute(context, path="test.txt")

        assert result.success is True
        assert result.output == "Hello, World!"

    async def test_read_file_not_found(self, tmp_path: Path):
        """测试文件不存在"""
        tool = ReadFileTool()
        context = ToolContext(working_directory=str(tmp_path))

        result = await tool.execute(context, path="nonexistent.txt")

        assert result.success is False
        assert "not found" in result.error
```

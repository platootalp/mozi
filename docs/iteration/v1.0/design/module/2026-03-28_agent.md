## 1. 模块概述

Agent 模块是系统的执行核心，负责实际的任务执行。每个 Agent 是一个独立的执行单元，具备自己的思考和执行能力。Agent 层不追求分层结构，本质是模块串连协作。

### 1.1 核心职责

- 任务执行 (think + act)
- 工具调用
- 结果返回
- 状态管理

### 1.2 差异化特性

| 特性 | 说明 |
|------|------|
| ReAct 循环 | Reasoning + Acting 模式 |
| 工具动态绑定 | 每个 Agent 可绑定不同工具 |
| 可扩展 | 支持自定义 Agent |

## 3. BaseAgent 抽象基类

> **类型定义说明**: `Thought`、`Result`、`AgentRunResult`、`AgentState` 等核心类型定义在 [2026-03-28_shared_types.md](./2026-03-28_shared_types.md) 中，本文档引用这些类型不再重复定义。

### 3.1 接口定义

```python
class BaseAgent(ABC):
    """Agent 基类，所有 Agent 必须继承此类"""

    @property
    @abstractmethod
    def name(self) -> str:
        """Agent 名称"""
        pass

    @property
    @abstractmethod
    def description(self) -> str:
        """Agent 描述"""
        pass

    @property
    def tools(self) -> List[Tool]:
        """Agent 可用的工具列表"""
        return []

    @property
    def supported_intents(self) -> List[Intent]:
        """支持的意图类型"""
        return []

    @abstractmethod
    async def think(self, task: Task, context: Context) -> Thought:
        """
        思考阶段：分析任务，决定行动
        - 输入：任务、上下文
        - 输出：思考结果（包含行动计划）
        """
        pass

    @abstractmethod
    async def execute(self, thought: Thought) -> Result:
        """
        执行阶段：执行思考决定的行动
        - 输入：思考结果
        - 输出：执行结果
        """
        pass

    async def run(self, task: Task, context: Context) -> AgentRunResult:
        """
        执行单次 ReAct 迭代（think + execute）

        注意：这是单次迭代，不是循环。
        多轮迭代由 Orchestration 层的 RalphLoop 负责。

        - think(): 分析任务，决定行动（LLM 调用）
        - execute(): 执行行动（工具调用）
        - 返回本次迭代结果，包含 progress 字段供 RalphLoop 判断
        """
        thought = await self.think(task, context)
        result = await self.execute(thought)
        return AgentRunResult(
            thought=thought,
            result=result,
            progress=self._calculate_progress(result)
        )

    def _calculate_progress(self, result: Result) -> float:
        """计算本次迭代的进度"""
        if getattr(result, 'is_complete', False):
            return 1.0
        return getattr(result, 'progress', 0.0)
```

## 5. 扩展 Agent

### 5.1 加载机制

从 `~/.mozi/agents/` 目录加载自定义 Agent。

```python
class AgentRegistry:
    def __init__(self):
        self._builtin_agents: Dict[str, BaseAgent] = {}
        self._external_agents: Dict[str, BaseAgent] = {}

    def load_builtin_agents(self) -> Dict[str, BaseAgent]:
        """加载内置 Agent"""
        return {
            "builder": BuilderAgent(),
            "reviewer": ReviewerAgent(),
            "explorer": ExplorerAgent(),
            "planner": PlannerAgent(),
            "researcher": ResearcherAgent(),
        }

    def load_external_agents(self) -> Dict[str, BaseAgent]:
        """从 ~/.mozi/agents/ 加载外部 Agent"""
        agents = {}
        agents_dir = Path("~/.mozi/agents/").expanduser()

        if not agents_dir.exists():
            return agents

        for path in agents_dir.glob("*.py"):
            if path.name.startswith("_"):
                continue
            agent = self._load_agent_module(path)
            if agent:
                agents[agent.name] = agent

        return agents

    def _load_agent_module(self, path: Path) -> Optional[BaseAgent]:
        """加载单个 Agent 模块"""
        # 安全校验：确保路径在 ~/.mozi/agents/ 目录下
        agents_dir = Path("~/.mozi/agents/").expanduser().resolve()
        resolved_path = path.resolve()
        if not str(resolved_path).startswith(str(agents_dir)):
            return None

        module_name = path.stem
        spec = importlib.util.spec_from_file_location(module_name, path)
        if not spec or not spec.loader:
            return None

        module = importlib.util.module_from_spec(spec)
        spec.loader.exec_module(module)

        for item in dir(module):
            obj = getattr(module, item)
            if isinstance(obj, type) and issubclass(obj, BaseAgent) and obj != BaseAgent:
                return obj()
        return None

    def get_agent(self, name: str) -> Optional[BaseAgent]:
        """获取 Agent 实例"""
        return self._builtin_agents.get(name) or self._external_agents.get(name)

    def list_agents(self) -> List[str]:
        """列出所有可用 Agent"""
        return list(self._builtin_agents.keys()) + list(self._external_agents.keys())
```

## 7. 核心工作流

### 7.1 Agent ReAct 流程（单次迭代）

```
任务输入
    │
    ▼
┌─────────────────────────────────────────┐
│              THINK (思考)                 │
│                                          │
│  1. 理解任务                             │
│  2. 检查上下文                           │
│  3. 决定行动                             │
│  4. 生成工具调用计划                      │
└──────────────────┬──────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────┐
│              ACT (执行)                   │
│                                          │
│  1. 调用工具                             │
│  2. 获取结果                             │
│  3. 更新上下文                           │
└──────────────────┬──────────────────────┘
                   │
                   ▼
            返回 AgentRunResult
            (含 progress 字段)
```

**注意**: 这是单次迭代流程图。多轮迭代由 RalphLoop 控制。

### 7.2 Agent 迭代与 RalphLoop 的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    RalphLoop (Orchestration)                │
│                                                             │
│  while iteration < max_iterations:                         │
│      │                                                      │
│      ▼                                                      │
│  ┌───────────────────────────────────────────────────────┐ │
│  │  agent.run(task, context)  ← 单次 ReAct 迭代           │ │
│  │                                                       │ │
│  │  ├── think() → Thought (LLM 分析)                     │ │
│  │  └── execute(thought) → Result                         │ │
│  │                        ↓                               │ │
│  │                   AgentRunResult                       │ │
│  │                   (含 progress)                         │ │
│  └───────────────────────────────────────────────────────┘ │
│      │                                                      │
│      ▼                                                      │
│  if progress >= 0.95:  ← 任务完成？                        │
│      break                                                      │
│                                                             │
│  if is_stuck(results):  ← 卡死检测                         │
│      todo_enforcer.reenact(task)                          │
└─────────────────────────────────────────────────────────────┘
```

**职责边界**:
- **Agent.run()**: 单次 think + execute，返回 progress
- **RalphLoop**: 多轮迭代 + 进度检测 + 卡死恢复
- **ExecutionStrategy**: 定义迭代次数上限 (SIMPLE=5, MEDIUM=15, COMPLEX=30)

## 9. Claude Code Agent Alignment

本节分析 Claude Code 的 Agent 架构与 Mozi 的多智能体架构的差异，并提供对齐建议。

### 9.1 架构对比

| 维度 | Claude Code | Mozi |
|------|-------------|------|
| Agent 架构 | 单体 Agent + 模式切换 | 多 Agent 专业化分工 |
| 模式切换 | `/explore`, `/plan`, 默认模式 | 通过意图识别路由到专用 Agent |
| 上下文管理 | 隐式管理 | 显式复杂度评估与路由 |
| 工具集 | 内置 6 个工具 (bash, edit, glob, grep, read, write) | 可动态绑定的工具注册表 |
| 权限系统 | Agent 级别集成 | 分层权限管理 |

### 9.2 Claude Code 模式分析

#### Explore 模式 (`/explore`)
- **特性**: 深度代码库探索，无编辑能力
- **工具限制**: 只读操作 (read, glob, grep)
- **适用场景**: 代码库结构分析、依赖追踪、架构理解

#### Plan 模式 (`/plan`)
- **特性**: 审查代码并提出变更建议，纯只读
- **工具限制**: 只读操作
- **适用场景**: PR 审查、代码质量评估、变更建议

#### General 模式 (默认)
- **特性**: 完整读写执行能力
- **工具**: 全部内置工具
- **适用场景**: 代码编写、修改、执行命令

### 9.3 Mozi AgentMode 枚举

```python
from enum import Enum, auto

class AgentMode(Enum):
    """Agent 运行模式，对齐 Claude Code 模式切换"""
    EXPLORE = auto()   # 探索模式 - 只读，深度分析
    PLAN = auto()     # 计划模式 - 只读，审查与建议
    GENERAL = auto()  # 一般模式 - 读写执行
```

### 9.4 对齐建议

#### 9.4.1 Explorer Agent 对齐 Explore 模式

Mozi 的 Explorer Agent 已与 Claude Code Explore 模式对齐：

| 特性 | Claude Code `/explore` | Mozi Explorer Agent |
|------|------------------------|---------------------|
| 工具集 | read, glob, grep | read, glob, grep |
| 编辑能力 | 无 | 无 |
| 上下文限制 | 会话级 | 任务级 |
| 适用任务 | 代码库探索、依赖分析 | 结构分析、关系发现 |

**推荐增强**:
- 添加结构化输出格式，支持生成调用图、依赖矩阵
- 增加探索深度控制参数

#### 9.4.2 Planner Agent 对齐 Plan 模式

Mozi 的 Planner Agent 可进一步对齐 Claude Code Plan 模式：

| 特性 | Claude Code `/plan` | Mozi Planner Agent |
|------|---------------------|---------------------|
| 工具集 | read, glob, grep | read, glob, grep |
| 写能力 | 无 | 可选 |
| 输出 | 变更建议 | 任务分解 + 执行计划 |

**推荐增强**:
- 添加 `read_only` 模式标志，与 Claude Code Plan 模式一致
- 支持生成结构化变更建议而非直接修改

#### 9.4.3 Builder Agent 对齐 General 模式

| 特性 | Claude Code 默认模式 | Mozi Builder Agent |
|------|---------------------|-------------------|
| 工具集 | 全部 6 个 | 全部内置工具 |
| 写能力 | 完整 | 完整 |
| 执行能力 | bash | bash |

### 9.5 架构融合建议

#### 方案 A: 保持多 Agent 架构 (推荐)

保持 Mozi 的多 Agent 专业分工，通过意图识别自动路由：

```
用户输入 → 意图识别 → Agent 路由
                          │
    ┌──────────────────────┼──────────────────────┐
    │                      │                      │
    ▼                      ▼                      ▼
 EXPLORE               PLAN                    GENERAL
 (Explorer)           (Planner)               (Builder)
```

优点:
- 专业分工，单一职责
- 可并行执行独立任务
- 易于扩展新 Agent 类型

#### 方案 B: 引入 AgentMode 的单体 Agent

如需与 Claude Code 更紧密对齐，可引入 AgentMode：

```python
class MoziAgent:
    def __init__(self, mode: AgentMode = AgentMode.GENERAL):
        self._mode = mode
        self._tools = self._get_tools_for_mode(mode)

    def _get_tools_for_mode(self, mode: AgentMode) -> List[Tool]:
        if mode == AgentMode.EXPLORE:
            return [ReadTool(), GlobTool(), GrepTool()]
        elif mode == AgentMode.PLAN:
            return [ReadTool(), GlobTool(), GrepTool()]
        else:  # GENERAL
            return ALL_TOOLS
```

适用场景:
- CLI 入口与 Claude Code 对齐
- 简化 Agent 管理复杂度

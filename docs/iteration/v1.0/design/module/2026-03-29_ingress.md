# Ingress 模块设计文档

## 1. 模块概述

### 1.1 模块名称

Ingress（接入层）

### 1.2 职责

Ingress 模块是 Mozi 系统的用户入口，负责接收用户输入、解析命令、管理 REPL 交互循环，并将请求路由到编排层（Orchestration）。

### 1.3 核心能力

| 能力         | 说明                                                         |
| ------------ | ------------------------------------------------------------ |
| REPL 交互    | 支持交互式对话循环，实时响应用户输入                         |
| 命令模式     | 支持离线命令执行（`mozi do "任务"`）                         |
| 会话管理     | 支持创建、继续、列表、删除会话                               |
| 输出格式化   | 结构化输出、错误提示、进度展示                               |
| 事件发布     | 通过 EventBus 将用户消息发布到编排层                         |

### 1.4 核心问题

| 问题 | 说明 |
| ---- | ---- |
| 用户输入多样化 | 支持交互式 REPL 和离线命令两种模式 |
| 输出格式化 | 如何结构化展示执行结果、错误信息 |
| 会话管理入口 | 作为用户与系统交互的入口，需要管理会话生命周期 |
| 命令解析 | 解析用户输入的命令和参数 |

### 1.5 难点

| 难点 | 说明 |
| ---- | ---- |
| REPL 状态管理 | 维护交互状态，支持多会话切换 |
| 错误输出友好 | 将系统错误转换为用户友好提示 |
| 输出性能 | 大量输出时的性能问题 |
| 历史命令 | 支持命令历史和补全 |

### 1.6 解决方案

| 方案 | 说明 |
| ---- | ---- |
| ReplCLI + CommandCLI | 双入口设计，分别处理交互和离线场景 |
| 输出格式化器 | 结构化输出、颜色区分、进度条 |
| EventBus 集成 | 通过事件总线与编排层解耦通信 |
| 命令历史 | 支持上下键查看历史命令 |

## 3. 接口设计

### 3.1 核心类型定义

```python
from dataclasses import dataclass
from enum import Enum
from typing import Optional


class CliMode(Enum):
    """CLI 运行模式"""
    REPL = "repl"       # 交互式 REPL 模式
    COMMAND = "command" # 单次命令模式


class SessionAction(Enum):
    """会话操作类型"""
    CREATE = "create"
    CONTINUE = "continue"
    LIST = "list"
    DELETE = "delete"


@dataclass
class UserInput:
    """用户输入结构"""
    mode: CliMode
    content: str
    session_id: Optional[str] = None
    verbose: bool = False


@dataclass
class CliResult:
    """CLI 执行结果"""
    success: bool
    content: str
    session_id: Optional[str] = None
    error: Optional[str] = None
```

### 3.2 核心类定义

```python
from abc import ABC, abstractmethod
from typing import AsyncIterator

from mozi.cli.output import OutputFormatter
from mozi.core.error import MoziError


class BaseCLI(ABC):
    """CLI 抽象基类"""

    def __init__(self, output: OutputFormatter) -> None:
        self._output = output

    @abstractmethod
    async def run(self, input_data: UserInput) -> CliResult:
        """执行 CLI 命令"""
        ...


class ReplCLI(BaseCLI):
    """REPL 交互式 CLI 实现"""

    def __init__(
        self,
        output: OutputFormatter,
        event_bus: "EventBus",
        session_manager: "SessionManager",
    ) -> None:
        super().__init__(output)
        self._event_bus = event_bus
        self._session_manager = session_manager
        self._running: bool = False

    async def run(self, input_data: UserInput) -> CliResult:
        """启动 REPL 循环"""
        self._running = True
        session = await self._session_manager.create()
        try:
            while self._running:
                line = await self._read_line()
                if line is None:
                    break
                result = await self._process_line(line, session.id)
                self._output.print(result)
            return CliResult(success=True, content="Session ended")
        except MoziError as e:
            return CliResult(success=False, content="", error=str(e))

    async def _read_line(self) -> Optional[str]:
        """读取用户输入行"""
        ...

    async def _process_line(self, line: str, session_id: str) -> str:
        """处理单行输入"""
        await self._event_bus.publish(
            "user_message",
            {"session_id": session_id, "content": line},
        )
        # 等待并返回响应
        ...

    def stop(self) -> None:
        """停止 REPL 循环"""
        self._running = False


class CommandCLI(BaseCLI):
    """命令模式 CLI 实现"""

    def __init__(
        self,
        output: OutputFormatter,
        event_bus: "EventBus",
        session_manager: "SessionManager",
    ) -> None:
        super().__init__(output)
        self._event_bus = event_bus
        self._session_manager = session_manager

    async def run(self, input_data: UserInput) -> CliResult:
        """执行单次命令"""
        if input_data.session_id:
            session_id = input_data.session_id
        else:
            session = await self._session_manager.create()
            session_id = session.id

        await self._event_bus.publish(
            "user_message",
            {
                "session_id": session_id,
                "content": input_data.content,
                "verbose": input_data.verbose,
            },
        )
        # 等待并返回响应
        result = await self._wait_for_result(session_id)
        return CliResult(
            success=result.get("success", False),
            content=result.get("content", ""),
            session_id=session_id,
            error=result.get("error"),
        )


class SessionManager:
    """会话管理器接口"""

    async def create(self) -> "Session":
        """创建新会话"""
        ...

    async def get(self, session_id: str) -> Optional["Session"]:
        """获取会话"""
        ...

    async def list(self) -> list["Session"]:
        """列出所有会话"""
        ...

    async def delete(self, session_id: str) -> bool:
        """删除会话"""
        ...


class OutputFormatter:
    """输出格式化器"""

    def print(self, result: CliResult) -> None:
        """打印结果"""
        ...

    def print_error(self, error: str) -> None:
        """打印错误"""
        ...

    def print_progress(self, message: str, progress: float) -> None:
        """打印进度"""
        ...


class EventBus:
    """事件总线接口（由基础设施层提供）"""

    async def publish(self, event: str, payload: dict) -> None:
        """发布事件"""
        ...

    async def subscribe(
        self, event: str, handler: callable
    ) -> AsyncIterator[dict]:
        """订阅事件"""
        ...
```

## 4. 数据流

### 4.1 整体数据流

```
用户输入
    │
    ▼
CLI 解析
    │
    ├──► REPL 模式 ──► 创建/复用会话 ──► EventBus.publish("user_message")
    │
    └──► 命令模式 ──► 创建会话 ──► EventBus.publish("user_message")
                              │
                              ▼
                        Orchestrator（订阅事件）
                              │
                              ▼
                        CLI 输出结果
```

### 4.2 与上下层交互

| 层级           | 交互方向 | 接口                         | 说明                    |
| -------------- | -------- | ---------------------------- | ----------------------- |
| 编排层（Orchestration）| 下游     | EventBus.publish("user_message") | 发布用户消息事件        |
| 编排层（Orchestration）| 上游     | EventBus.subscribe("agent_response") | 订阅 Agent 响应事件     |
| 基础设施层（Infrastructure）| 依赖     | EventBus                     | 事件发布/订阅           |
| 基础设施层（Infrastructure）| 依赖     | SessionManager               | 会话生命周期管理        |
| 基础设施层（Infrastructure）| 依赖     | ConfigLoader                 | 配置加载                |

### 4.3 EventBus 事件定义

| 事件名             | 方向    | Payload                                    | 说明           |
| ------------------ | ------- | ------------------------------------------ | -------------- |
| `user_message`     | Ingress -> Orchestration | `{"session_id", "content", "verbose"}` | 用户消息       |
| `agent_response`   | Orchestration -> Ingress | `{"session_id", "content", "success"}` | Agent 响应     |
| `tool_execution`   | Orchestration -> Ingress | `{"tool", "args", "result"}`          | 工具执行结果   |
| `session_created`  | Orchestration -> Ingress | `{"session_id"}`                        | 会话创建事件   |
| `error`            | Orchestration -> Ingress | `{"session_id", "error", "code"}`      | 错误事件       |

### 6.3 测试用例示例

```python
import pytest
from mozi.cli.commands import parse_command, CliMode


class TestParseCommand:
    """命令解析测试"""

    def test_parse_repl_command(self) -> None:
        """测试 REPL 模式解析"""
        result = parse_command(["mozi"])
        assert result.mode == CliMode.REPL

    def test_parse_do_command(self) -> None:
        """测试 do 命令解析"""
        result = parse_command(["mozi", "do", "write tests"])
        assert result.mode == CliMode.COMMAND
        assert result.content == "write tests"

    def test_parse_invalid_command(self) -> None:
        """测试无效命令"""
        with pytest.raises(CommandParseError):
            parse_command(["mozi", "invalid"])

    def test_parse_session_command(self) -> None:
        """测试会话命令解析"""
        result = parse_command(["mozi", "session", "list"])
        assert result.session_action == SessionAction.LIST
```

## 8. 安全考虑

| 考虑项           | 实现方式                                      |
| ---------------- | --------------------------------------------- |
| 输入验证         | 对用户输入进行长度限制和字符过滤              |
| 敏感信息保护     | 不在日志中打印会话内容                        |
| 命令注入防护     | 使用列表参数传递命令，不使用 shell=True       |
| 超时保护         | 所有操作设置超时限制，防止无限等待            |

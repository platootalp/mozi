## 1. 项目愿景与定位

### 1.1 核心目标

**本地优先的 AI Coding Agent**，通过自适应复杂度路由和差异化记忆/安全模型提供智能编程辅助。

### 1.2 差异化定位

| 维度 | Mozi | Claude Code | 差异点 |
|------|------|-------------|--------|
| **记忆系统** | 三层分离 + 智能压缩 + 用户可控 | CLAUDE.md + Auto Memory | Working→Long-term→Short-term 自动分层 |
| **安全模型** | 工具名单 + HITL + 沙箱 | 权限模式 | 命令级通配符 + 无复杂审批流 |
| **多 Agent** | 中心化 Orchestrator + 5 内置子 Agent | Subagents | Sisyphus 风格编排（灵感来自 oh-my-opencode） |
| **部署** | 本地优先 | 云端为主 | 数据不出本地 |

### 1.3 核心架构决策

| 决策点 | 选择 | 说明 |
|--------|------|------|
| Agent 架构 | **中心化** | Orchestrator 是唯一决策者，子 Agent 只执行 |
| 子 Agent 类型 | **5 个内置** | Builder + Reviewer + Explorer + Planner + Researcher |
| 会话/记忆关系 | **完全分离** | Session 管对话，Memory 管跨会话知识 |
| 安全模型 | **名单 + HITL + 沙箱** | 命令级通配符，无复杂审批流 |
| 工具执行 | **同步阻塞** | Agent 调用工具 → 等待结果 → 继续 |
| 记忆优先级 | **Working → Long-term → Short-term** | 上下文质量优先 |

### 各层职责边界

| 层级 | 职责 | 边界 |
|------|------|------|
| **Ingress** | CLI/Web/IDE 入口，命令解析，响应渲染 | 不涉及业务逻辑 |
| **Orchestrator** | 意图识别、复杂度评估、任务路由、子 Agent 调度 | 仅做决策，不直接操作文件 |
| **Capabilities** | 工具注册、权限检查、沙箱、HITL、Skills | 所有外部操作经由此层 |
| **Infrastructure** | 会话存储（与记忆分离）、记忆存储、模型调用 | 数据持久化 |

## Phase 2: 能力层

### 2.1 目标

实现工具框架和安全模型：
- 6 个内置工具（read/write/edit/bash/grep/glob）
- 命令级工具名单 + 通配符模式
- 沙箱隔离
- HITL 触发机制
- Skills 引擎基础

### 2.2 安全模型设计

```
┌─────────────────────────────────────────────────────────────────┐
│                        安全模型层级                              │
├─────────────────────────────────────────────────────────────────┤
│  Layer 1: 工具名单 (tools.json)                                 │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ deny:  ["rm -rf *", "git push --force", "curl *"]         │ │
│  │ ask:   ["bash:*", "write:*"]                              │ │
│  │ allow: ["read:*", "grep:*", "glob:*"]                     │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                           ↓                                      │
│  Layer 2: 沙箱隔离 (sandbox mode)                               │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ off:      无隔离                                            │ │
│  │ non-main: 禁止访问主目录 (~/)                               │ │
│  │ all:      禁止所有敏感路径                                  │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                           ↓                                      │
│  Layer 3: HITL 触发                                              │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 触发条件: ask 级别工具 + 命令匹配模式                        │ │
│  │ 处理流程: 暂停 → 用户确认 → 执行/拒绝                       │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                           ↓                                      │
│  Layer 4: 审计日志                                               │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 记录: 时间、用户、工具、命令、结果                          │ │
│  │ 存储: SQLite audit_logs 表                                 │ │
│  └─────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 工具名单通配符规则

| 模式 | 匹配示例 | 非匹配示例 |
|------|----------|------------|
| `bash:git push*` | `git push origin main` | `git pull` |
| `bash:rm -rf *` | `rm -rf /tmp/*` | `rm /tmp/file` |
| `bash:*` | 所有 bash 命令 | - |
| `write:*.py` | 任何 .py 文件写入 | `read:*.py` |
| `mcp:*` | 所有 MCP 工具 | - |

## Phase 3: 记忆系统

### 3.1 目标

实现三层记忆架构（完全独立于 Session）：
- **Working Memory**: LLM 上下文滑动窗口 + 上下文压缩
- **Long-term Memory**: SQLite 持久化（用户偏好、项目知识）
- **Short-term Memory**: 向量检索（RAG，Qdrant）

### 3.2 记忆架构设计

```
┌─────────────────────────────────────────────────────────────────┐
│                       记忆系统架构                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                   Working Memory                        │   │
│   │                   (LLM Context)                        │   │
│   ├─────────────────────────────────────────────────────────┤   │
│   │  滑动窗口策略:                                          │   │
│   │  - 保留最近 N 轮对话（可配置，默认 20）                  │   │
│   │  - 超出时早期消息移出或压缩摘要                         │   │
│   │                                                         │   │
│   │  上下文压缩:                                            │   │
│   │  - Token 达到 80% 阈值时触发                           │   │
│   │  - LLM 生成摘要，释放上下文空间                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                            ↓                                     │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  Long-term Memory                       │   │
│   │                  (SQLite + 文件系统)                    │   │
│   ├─────────────────────────────────────────────────────────┤   │
│   │  存储内容:                                              │   │
│   │  - 用户偏好（~/.mozi/memory/preferences.md）           │   │
│   │  - 项目知识（项目 .mozi/memory/）                      │   │
│   │  - 历史经验（跨会话学习到的模式）                       │   │
│   │                                                         │   │
│   │  召回方式:                                              │   │
│   │  - memory:recall <query> 工具按需加载                  │   │
│   │  - 自动召回（基于当前任务上下文相关）                  │   │
│   └─────────────────────────────────────────────────────────┘   │
│                            ↓                                     │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                  Short-term Memory                      │   │
│   │                  (Qdrant 向量库)                        │   │
│   ├─────────────────────────────────────────────────────────┤   │
│   │  存储内容:                                              │   │
│   │  - 代码片段嵌入（session 内）                           │   │
│   │  - 对话摘要向量                                         │   │
│   │  - 任务相关上下文                                       │   │
│   │                                                         │   │
│   │  检索方式:                                              │   │
│   │  - 语义相似度搜索                                       │   │
│   │  - BM25 关键词检索（降级方案）                          │   │
│   │  - 混合检索（语义 + 关键词）                            │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 记忆与会话的分离设计

```
┌────────────────────────────────┐     ┌────────────────────────────────┐
│         Session Manager        │     │        Memory System           │
│        (会话生命周期)           │     │       (跨会话知识)              │
├────────────────────────────────┤     ├────────────────────────────────┤
│  职责:                         │     │  职责:                         │
│  - 管理 CLI 对话生命周期        │     │  - 跨会话持久化知识             │
│  - 存储消息历史                 │     │  - 语义搜索和召回               │
│  - 跟踪当前任务状态             │     │  - 用户偏好管理                 │
│  - Checkpoint 管理             │     │  - RAG 检索                     │
│                                │     │                                │
│  存储:                         │     │  存储:                         │
│  - SQLite sessions 表          │     │  - SQLite (Long-term)          │
│  - SQLite messages 表          │     │  - Qdrant (Short-term)         │
│  - SQLite checkpoints 表       │     │  - 文件系统 (偏好/知识)         │
│                                │     │                                │
│  访问:                         │     │  访问:                         │
│  - Agent 在执行时创建/更新      │     │  - Orchestrator 按需调用       │
│  - 用户不可直接查看/编辑        │     │  - 用户可通过 memory:recall     │
│                                │     │                                │
│  生命周期:                     │     │  生命周期:                     │
│  - 会话结束 → 归档/删除        │     │  - 独立于会话，永久保留         │
│  - 与记忆无关                  │     │  - 按策略自动清理              │
└────────────────────────────────┘     └────────────────────────────────┘
```

### 3.4 层级转换策略

| 转换方向 | 触发条件 | 动作 |
|----------|----------|------|
| Working → Short-term | 会话结束 或 Token 达到 95% | 当前上下文摘要存入向量库 |
| Short-term → Long-term | 同一内容被召回 ≥3 次 | 标记为重要，持久化到文件系统 |
| Long-term → Short-term | 用户明确保存重要记忆 | 嵌入向量库供语义检索 |
| Short-term → 遗忘 | 180 天未被召回 | 自动删除 |
| Long-term → 遗忘 | 用户手动删除 | 永久删除 |

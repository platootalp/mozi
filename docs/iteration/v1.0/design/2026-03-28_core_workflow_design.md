## 1. 概述

本文档描述 Mozi Agent 从接收用户需求到完成代码编写的核心工作流程。

**核心原则**：
- 任务 100% 完成（RalphLoop 保证）
- 安全优先（每次工具调用都经过安全检查）
- 本地优先（数据不离开本地环境）

## 3. 第一阶段：需求理解

### 3.1 意图识别 (Intent Recognition)

> **核心原则**：Intent Recognition 识别**宽泛意图类别**，然后路由给对应 Agent。Agent 内部自主决定具体执行步骤，而不是由 LLM 决定具体任务。

```
用户输入: "帮我添加一个登录功能"
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      IntentGate (LLM 推理)                       │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ 输入 Prompt:                                            │  │
│  │                                                         │  │
│  │ "识别用户意图类别（只返回宽泛类别）:                     │  │
│  │  - EXPLORE: 探索代码库                                 │  │
│  │  - CODE: 编写/修改代码                                 │  │
│  │  - REVIEW: 审查代码                                    │  │
│  │  - PLAN: 规划任务                                       │  │
│  │  - RESEARCH: 研究技术                                   │  │
│  │                                                         │  │
│  │ 用户输入: {user_input}                                  │  │
│  │ 只返回最合适的宽泛类别。"                               │  │
│  └─────────────────────────────────────────────────────────┘  │
│                              │                                 │
│                              ▼                                 │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ LLM 推理输出:                                          │  │
│  │                                                         │  │
│  │ IntentResult(                                           │  │
│  │     intent=EXPLORE,  # 宽泛类别，不是具体任务         │  │
│  │     confidence=0.95                                     │  │
│  │ )                                                      │  │
│  │                                                         │  │
│  │ 解释: 用户想要编写代码，路由给 Builder Agent            │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 宽泛意图类型

| 意图 | 描述 | 路由到的 Agent |
|------|------|----------------|
| **EXPLORE** | 探索代码库、了解项目结构 | Explorer Agent |
| **CODE** | 编写/修改代码 | Builder Agent |
| **REVIEW** | 审查代码 | Reviewer Agent |
| **PLAN** | 规划复杂任务 | Planner Agent |
| **RESEARCH** | 研究技术方案 | Researcher Agent |

### 3.4 Sub-Agent 自主执行

> **关键点**：Sub-Agent 接收宽泛指令后，内部自主决定具体执行步骤。

```
Builder Agent 接收: "添加一个登录功能"
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      Builder Agent (自主决策)                     │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ Agent 内部流程:                                        │  │
│  │                                                         │  │
│  │ 1. 探索项目结构 (调用 read/glob/grep)                 │  │
│  │ 2. 分析需要创建的文件                                   │  │
│  │ 3. 创建路由/控制器/视图                                │  │
│  │ 4. 验证代码正确性                                      │  │
│  │ 5. 返回执行结果                                        │  │
│  │                                                         │  │
│  │ 不需要外部告诉 Agent 每一步做什么，                    │  │
│  │ 由 Agent 根据上下文自主决定。                          │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 第三阶段：ReAct 执行循环

### 5.1 RalphLoop 核心逻辑

```
┌─────────────────────────────────────────────────────────────────┐
│                      RalphLoop                                  │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │                                                         │  │
│  │  async def execute(task: Task) -> LoopResult:         │  │
│  │                                                         │  │
│  │      iteration = 0                                     │  │
│  │      while not task.is_complete:                       │  │
│  │                                                         │  │
│  │          # 1. Reason (思考)                           │  │
│  │          thought = await agent.think(task)             │  │
│  │                                                         │  │
│  │          # 2. Act (执行)                              │  │
│  │          if thought.needs_tool:                       │  │
│  │              result = await execute_tool(thought)      │  │
│  │          else:                                         │  │
│  │              result = await model.respond(thought)    │  │
│  │                                                         │  │
│  │          # 3. Observe (观察)                          │  │
│  │          observation = process_result(result)          │  │
│  │          task.add_to_context(observation)             │  │
│  │                                                         │  │
│  │          # 4. Check Progress                          │  │
│  │          if task.progress >= 0.95:                     │  │
│  │              task.is_complete = True                   │  │
│  │                                                         │  │
│  │          # 5. Stuck Detection                         │  │
│  │          if self._is_stuck(thoughts):                 │  │
│  │              await self._reenact(task)                 │  │
│  │                                                         │  │
│  │          iteration += 1                                 │  │
│  │          if iteration >= max_iterations:               │  │
│  │              raise MaxIterationsError()                │  │
│  │                                                         │  │
│  │      return LoopResult(state=COMPLETED)                 │  │
│  │                                                         │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 5.3 Stuck Detection & Reenact

```
当连续 3 次思考结果相似时，触发 Stuck Detection:

┌─────────────────────────────────────────────────────────────────┐
│                      TodoEnforcer                               │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ def _is_stuck(self, thoughts: List[Thought]) -> bool: │  │
│  │                                                         │  │
│  │      if len(thoughts) < 3:                             │  │
│  │          return False                                  │  │
│  │                                                         │  │
│  │      # 检查最近 3 次思考是否重复                        │  │
│  │      recent = thoughts[-3:]                            │  │
│  │      if all(t.tool == recent[0].tool for t in recent):│  │
│  │          if all(t.params == recent[0].params ...):    │  │
│  │              return True  # stuck!                     │  │
│  │                                                         │  │
│  │      return False                                       │  │
│  └─────────────────────────────────────────────────────────┘  │
│                              │                                 │
│                              ▼                                 │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ _reenact(task):                                        │  │
│  │      # 清空当前上下文，重新开始                         │  │
│  │      task.clear_context()                             │  │
│  │      task.set_state(INITIAL)                           │  │
│  │      # 从上一次的检查点恢复                             │  │
│  │      task.restore_checkpoint()                         │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

## 7. 第五阶段：上下文管理

### 7.1 Context 组装

```
                              ▼

│  ┌─────────────────────────────────────────────────────────┐  │
```

## 9. 错误处理与恢复

### 9.1 错误分类

| 错误类型 | 说明 | 处理策略 |
|----------|------|----------|
| `ToolError` | 工具执行失败 | 重试 3 次，失败则回退 |
| `ModelError` | 模型调用失败 | 重试 3 次，指数退避 |
| `PermissionError` | 权限不足 | 询问用户授权 |
| `TimeoutError` | 执行超时 | 重试或跳过 |
| `SyntaxError` | 代码语法错误 | 标记错误，继续其他任务 |

### 9.2 恢复流程

```
任务执行中遇到错误:
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│                      ErrorHandler                               │
│                                                               │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ 1. 错误分类                                            │  │
│  │      error.type = "ToolError"                          │  │
│  │      error.detail = "bash timeout"                     │  │
│  └─────────────────────────────────────────────────────────┘  │
│                              │                                 │
│                              ▼                                 │
│  ┌─────────────────────────────────────────────────────────┐  │
│  │ 2. 恢复决策                                            │  │
│  │                                                         │  │
│  │      if retry_count < MAX_RETRIES:                     │  │
│  │          retry()  # 指数退避                           │  │
│  │      elif can_skip(error):                             │  │
│  │          skip()  # 可跳过则继续                        │  │
│  │      else:                                             │  │
│  │          rollback()  # 回退到上一个检查点              │  │
│  │          ask_user()  # 询问用户                       │  │
│  └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 9.3 检查点与回退

```
每次工具执行成功后，创建检查点:

┌─────────────────────────────────────────────────────────────────┐
│                      CheckpointManager                           │
│                                                               │
│  checkpoint = {                                                │
│      "iteration": 3,                                          │
│      "task_state": {...},                                     │
│      "modified_files": ["a.py", "b.py"],                      │
│      "git_status": "dirty"                                    │
│  }                                                             │
│                                                               │
│  save_checkpoint(checkpoint)                                   │
│                                                               │
│  回退时:                                                       │
│  1. git checkout -- <modified_files>  # 恢复文件              │
│  2. restore task_state from checkpoint                        │
│  3. resume from last successful tool                          │
└─────────────────────────────────────────────────────────────────┘
```

## 11. 术语表

| 术语 | 说明 |
|------|------|
| **Intent Recognition** | 意图识别，解析用户输入确定任务类型 |
| **Complexity Assessment** | 复杂度评估，计算任务复杂度得分 |
| **RalphLoop** | 自循环执行器，确保任务 100% 完成 |
| **ReAct** | Reasoning + Acting，思考与执行交替的循环模式 |
| **Tool Executor** | 工具执行器，负责调用和管理工具 |
| **Context Assembler** | 上下文组装器，管理 Prompt 组成 |
| **HITL** | Human-in-the-Loop，人工确认机制 |
| **Checkpoint** | 检查点，保存任务状态用于恢复 |

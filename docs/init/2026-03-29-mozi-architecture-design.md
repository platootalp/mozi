# Mozi Coding Agent 架构设计

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                              1. 接入层                                      │
│                    CLI (REPL)  │  IDE Plugin  │  REST API  │  WebSocket    │
└─────────────────────────────────┬───────────────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────────────┐
│                              2. 会话层                                      │
│            会话管理器：创建/销毁/路由 │ 并发控制 │ 鉴权 │ 配额管理           │
│            (注：这是会话的"容器"，每个 Session 实例持有 Orchestrator)       │
└─────────────────────────────────┬───────────────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────────────┐
│                              3. 编排层                                       │
│              意图识别 │ Planner │ 路由策略 │ 多 Agent 协调                  │
│              (注：这是会话的"大脑"，负责逻辑决策，不直接存储数据)            │
└─────────────────────────────────┬───────────────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────────────┐
│                              4. Core 层                                      │
│        Model Gateway │ 工具注册中心 │ MCP 客户端 │ Skills                    │
│        (注：负责与外部世界交互，屏蔽底层差异)                               │
└─────────────────────────────────┬───────────────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────────────┐
│                              5. 知识层                                       │
│      短期上下文：滑动窗口 │ 长期记忆：向量库 │ 项目知识库：RAG              │
│      (注：这是会话的"海马体"，纯数据服务，被编排层调用)                     │
└─────────────────────────────────┬───────────────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────────────┐
│                            6. 基础设施层                                    │
│      存储：SQLite/PG │ 配置 │ 事件总线 │ 日志/监控 │ 加密                   │
└───────────────────────────────────────────────────────────────────────────┘

══════════════════════ 横切面 (Cross-Cutting) ══════════════════════
   [安全沙箱] (Tool Sandbox)  │  [可观测性] (Tracing/Metrics)  │  [熔断限流]
```

## 2. 数据流

### 2.1 主交互流程

```
用户输入
    │
    ▼
CLI（解析命令/REPL）
    │
    ▼
EventBus.publish("user_message", payload)
    │
    ▼
Orchestrator（接收事件）
    │
    ├──► Orchestrator 意图识别
    ├──► Orchestrator 多轮澄清（如需要）
    ├──► Orchestrator 复杂度评分
    │         │
    │         ▼
    │    复杂度 ≤40 ──► SIMPLE路由 ──► 单Agent执行
    │    复杂度 41-70 ──► MEDIUM路由 ──► 单Agent + 工具
    │    复杂度 >70 ──► COMPLEX路由 ──► 多Agent协作
    │
    ▼
Context.build() ──► 从 Memory 召回短期+长期记忆
    │
    ▼
Model.invoke() ──► 生成响应/工具调用
    │
    ▼
Tools.execute() ──► 安全检查 ──► 白名单验证 ──► 执行
    │
    ▼
EventBus.publish("tool_result", payload)
    │
    ▼
Orchestrator（处理结果）
    │
    ▼
Session.append() ──► 更新上下文
    │
    ▼
Memory.compact() ──► 必要时压缩上下文
    │
    ▼
CLI 输出
```

### 2.2 多Agent协作流程（COMPLEX任务）

```
Orchestrator 触发多Agent模式
    │
    ▼
Planner 分解任务
    │
    ├──► Agent-A 执行子任务-1
    ├──► Agent-B 执行子任务-2（可并行）
    └──► Agent-C 执行子任务-3
              │
              ▼
        各Agent通过EventBus通信
              │
              ▼
        Orchestrator 汇总结果
              │
              ▼
        验证完整性 ──► 通过 ──► 返回结果
                     │
                     ▼（失败）
              回滚或重规划
```

## 3. 模块功能

### 接入层

| 模块    | 职责                                                                                |
| ------- | ----------------------------------------------------------------------------------- |
| **CLI** | REPL 交互循环、命令模式（`mozi do "任务"`）、会话管理（创建/继续/列表）、输出格式化 |

### 会话层

| 模块             | 职责                                                      |
| ---------------- | --------------------------------------------------------- |
| **Session Manager** | 会话生命周期管理、创建/销毁/路由、并发控制、鉴权、配额管理 |

### 编排层

| 模块             | 职责                                                      |
| ---------------- | --------------------------------------------------------- |
| **Orchestrator** | 任务规划、多轮澄清、复杂度评分、单Agent/多Agent路由、结果汇总 |
| **Planner**      | 任务分解、依赖分析、进度跟踪、失败恢复、回滚决策（COMPLEX 模式）|

### Core 层

| 模块                | 职责                                                               |
| ------------------- | ------------------------------------------------------------------ |
| **Model Gateway**   | 多模型支持（Anthropic/OpenAI）、模型选择、Prompt 模板管理、调用网关 |
| **Tools Registry**  | 内置工具注册（Read/Write/Edit/Glob/Grep/Bash）、工具发现、调用路由 |
| **MCP Client**      | MCP 协议客户端、外部工具连接管理                                   |
| **Skills**          | 可复用 prompt 工作流（如"生成测试"、"代码审查"）                   |

### 知识层

| 模块        | 职责                                                                  |
| ----------- | --------------------------------------------------------------------- |
| **Context** | 短期上下文滑动窗口、历史消息管理                                      |
| **Memory**  | 长期记忆向量库、项目知识库 RAG、用户偏好、关键事实                    |

### 基础设施层

| 模块              | 职责                                      |
| ----------------- | ----------------------------------------- |
| **Storage**       | SQLite/PG 存储、会话持久化                |
| **Config**        | 配置加载、校验、环境变量注入、热更新       |
| **EventBus**      | 异步发布/订阅、事件路由、插件通信         |
| **Logging/Metrics** | 结构化日志、Metrics 指标                 |
| **Encryption**    | 敏感数据加密                              |

### 横切面

| 模块         | 职责                                            |
| ------------ | ----------------------------------------------- |
| **安全沙箱** | Tool Sandbox、权限模型、工具白名单（支持通配符） |
| **可观测性** | Tracing、Metrics、结构化日志                    |
| **稳定性**   | 熔断、限流、指数退避重试、超时控制              |

## 4. 层级关系说明

### 4.1 调用关系

**逻辑依赖链：** 会话层 → 编排层 → Core 层 → 知识层 → 基础设施层

**说明**：
- **会话层** 是生命周期容器，管理 Session 实例的创建和销毁
- **编排层** 是控制平面，负责决策，主动调用 Core 层
- **Core 层** 负责与外部世界交互，屏蔽底层差异
- **知识层** 是数据平面，负责存储，被编排层调用

### 4.2 生命周期 vs. 架构层级

| 维度 | 说明 |
|------|------|
| **运行时容器** | Session 实例持有 Orchestrator 实例，会话包裹编排 |
| **架构依赖** | 编排层依赖知识层的数据，但会话层位于编排层上方（生命周期视角） |

**注**：架构分层描述的是**逻辑依赖关系**（编排层 → 知识层），而非对象包含关系。会话层强调生命周期管理，是运行时容器。

### 4.3 各层职责边界

| 层级 | 职责 | 类比 |
|------|------|------|
| **会话层** | 会话生命周期管理、创建/销毁/路由、并发控制、鉴权 | 容器外壳 |
| **编排层** | 意图识别、路由决策、任务分解、多Agent协调 | 大脑 |
| **Core 层** | 模型调用、工具执行、MCP集成、Skills执行 | 手脚 |
| **知识层** | 短期上下文、长期记忆、项目知识库 | 海马体 |
| **基础设施层** | 存储、配置、事件总线、日志、安全 | 骨架 |

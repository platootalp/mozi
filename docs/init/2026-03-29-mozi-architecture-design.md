# Mozi Coding Agent 架构设计

## 1. 整体架构

```
┌─────────────────────────────────────────────────────────┐
│                      接入层 (Ingress)                      │
│                    CLI（REPL + 命令模式）                  │
└─────────────────────────┬───────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────┐
│                    编排层 (Orchestration)                  │
│         Orchestrator（意图识别│多轮澄清│复杂识别│路由）     │
└─────────────────────────┬───────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────┐
│                    核心层 (Core)                         │
│  Planner │ Session │ Tools │ Skills │ MCP │ Model Adapter │
└─────────────────────────┬───────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────┐
│                   上下文层 (Context)                      │
│              Memory（短期+长期）│ Context                  │
└─────────────────────────┬───────────────────────────────┘
                          │
┌─────────────────────────▼───────────────────────────────┐
│                   基础设施层 (Infrastructure)             │
│              Config Loader │ Storage │ EventBus          │
└─────────────────────────────────────────────────────────┘

══════════════════ 横切面 ══════════════════
       可观测性（Logging│Metrics│Tracing）
          稳定性（Rate Limit│熔断│重试│超时）
             安全（权限│沙箱│白名单）
```

## 2. 数据流

### 2.1 主交互流程

```
用户输入
    │
    ▼
CLI（解析命令/REPL）
    │
    ▼
EventBus.publish("user_message", payload)
    │
    ▼
Orchestrator（接收事件）
    │
    ├──► Orchestrator 意图识别
    ├──► Orchestrator 多轮澄清（如需要）
    ├──► Orchestrator 复杂度评分
    │         │
    │         ▼
    │    复杂度 ≤40 ──► SIMPLE路由 ──► 单Agent执行
    │    复杂度 41-70 ──► MEDIUM路由 ──► 单Agent + 工具
    │    复杂度 >70 ──► COMPLEX路由 ──► 多Agent协作
    │
    ▼
Context.build() ──► 从 Memory 召回短期+长期记忆
    │
    ▼
Model.invoke() ──► 生成响应/工具调用
    │
    ▼
Tools.execute() ──► 安全检查 ──► 白名单验证 ──► 执行
    │
    ▼
EventBus.publish("tool_result", payload)
    │
    ▼
Orchestrator（处理结果）
    │
    ▼
Session.append() ──► 更新上下文
    │
    ▼
Memory.compact() ──► 必要时压缩上下文
    │
    ▼
CLI 输出
```

### 2.2 多Agent协作流程（COMPLEX任务）

```
Orchestrator 触发多Agent模式
    │
    ▼
Planner 分解任务
    │
    ├──► Agent-A 执行子任务-1
    ├──► Agent-B 执行子任务-2（可并行）
    └──► Agent-C 执行子任务-3
              │
              ▼
        各Agent通过EventBus通信
              │
              ▼
        Orchestrator 汇总结果
              │
              ▼
        验证完整性 ──► 通过 ──► 返回结果
                     │
                     ▼（失败）
              回滚或重规划
```

## 3. 模块功能

### 接入层

| 模块 | 职责 |
|------|------|
| **CLI** | REPL 交互循环、命令模式（`mozi do "任务"`）、会话管理（创建/继续/列表）、输出格式化 |

### 编排层

| 模块 | 职责 |
|------|------|
| **Orchestrator** | 意图识别、多轮澄清、复杂度评分、单Agent/多Agent路由、结果汇总 |

### 核心层

| 模块 | 职责 |
|------|------|
| **Planner** | 任务分解、依赖分析、进度跟踪、失败恢复、回滚决策 |
| **Session Manager** | 会话生命周期管理、元数据存储、上下文窗口管理 |
| **Tools Registry** | 内置工具注册（Read/Write/Edit/Glob/Grep/Bash）、工具发现、调用路由 |
| **MCP Client** | MCP 协议客户端、外部工具连接管理 |
| **Skills** | 可复用 prompt 工作流（如"生成测试"、"代码审查"） |
| **Model Adapter** | 多模型支持（Anthropic/OpenAI）、模型选择、Prompt 模板管理 |

### 上下文层

| 模块 | 职责 |
|------|------|
| **Memory** | 短期：滑动窗口（最近N轮对话）<br>长期：用户偏好、关键事实（文件存储） |
| **Context** | 当前任务上下文构建、历史消息管理、向量化检索（长期记忆） |

### 基础设施层

| 模块 | 职责 |
|------|------|
| **Config Loader** | JSON 配置加载、校验、环境变量注入、热更新 |
| **Storage** | 会话持久化、长期记忆存储、文件管理 |
| **EventBus** | 异步发布/订阅、事件路由、插件通信 |

### 横切面

| 模块 | 职责 |
|------|------|
| **可观测性** | 结构化日志、Metrics 指标、分布式链路追踪 |
| **稳定性** | Rate Limit 限流、熔断器、指数退避重试、超时控制 |
| **安全** | 权限模型、工具白名单（支持通配符）、沙箱隔离 |

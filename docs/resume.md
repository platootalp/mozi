项目背景：为提升研发效能，构建可自托管的 AI Coding Agent，支持长会话管理、多步工具调用与多 Agent 协作；在任务规划、记忆机制、工具调用、上下文工程等核心模块形成一体化能力，作为公司 AI Coding 战略的基础设施，支撑需求->编码->测试的自动化闭环。
技术栈：Python、Pydantic、Langgraph、OpenAI API、FastMCP、SQLite、Milvus、Typer、Rich
核心工作：
•
任务规划与执行追踪：基于 LLM 实现意图识别与多轮澄清机制，自动识别需求模糊点并引导补充上下文；采用 DAG 建模任务依赖，支持静态拆解与执行期动态重组；基于 asyncio 异步状态机驱动任务流转，TodoList 显式追踪任务状态，集成指数退避重试、熔断降级等容错策略，任务成功率 90%+。
•
高可靠工具调用框架：构建标准化工具接入层，支持文件系统、Bash、AST-Grep、LSP、Web 检索等 10+ 工具可插拔扩展；实现哈希锚定编辑机制（LINE#ID + 内容哈希），解决多 Agent 并发编辑冲突；实现错误分类体系与指数退避重试机制，工具调用失败率从 15% 降至 3%；实现基于通配符的工具权限白名单，敏感操作触发 HITL 人工介入机制，确经用户授权后执行。
•
记忆存储与上下文工程：设计三级存储架构（内存会话窗口 + Milvus 项目向量库 + SQLite 结构化存储），实现热-温-冷数据分层管理；构建双轨检索策略（工作区内 grep/glob 精准检索+ 跨仓库语义混合向量检索），解决大规模仓库加载瓶颈；实现工具调用长上下文本地文件卸载与 Subagent 上下文隔离机制，长会话 Token 消耗降低 约 30%。
•
多 Agent 协作编排层：设计主从式分层调度架构，Orchestrator 基于任务语义特征（规划/探索/编码/Review）动态实例化专业 Subagent，通过上下文快照打包与状态隔离机制实现子任务委派，避免多 Agent 间状态污染；实施 Agent 差异化工具权限管控（如探索Agent无写权限），保障并行协作安全。

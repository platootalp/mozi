# 文档规范（强制）

## 1. 概述

本规范定义项目全生命周期文档的结构、命名和管理要求。文档分两类：基础文档（贯穿始终）和迭代文档（按版本管理）。

## 2. 目录结构

```
docs/
├── foundation/                    # 🌟 贯穿始终的基础文档
│   ├── architecture/              # 架构设计
│   ├── requirement/               # 需求文档（PRD）
│   ├── design/                   # 系统设计
│   ├── api/                      # API 契约
│   └── data/                     # 数据模型
│
├── iteration/                     # 📦 迭代版本（每个版本独立）
│   ├── v1.0/
│   │   ├── README.md             # 版本整体说明（渐进式描述）
│   │   ├── prd/                  # 需求文档
│   │   ├── design/               # 设计文档
│   │   ├── development/          # 开发文档（技术方案等）
│   │   ├── testing/              # 测试报告
│   │   ├── release/              # 发布文档
│   │   └── retrospective/        # 复盘报告
│   ├── v1.1/
│   │   └── ...
│   └── v1.2/
│       └── ...
│
└── support/                   # 支撑体系（贯穿全周期）
    ├── adr/                      # 架构决策记录
    ├── sop/                      # 标准操作流程
    └── metric/                   # 度量指标定义
```

## 3. 命名规则

| 类型     | 规则               | 示例                                |
| -------- | ------------------ | ----------------------------------- |
| 基础文档 | `时间_文档名称.md` | `2026-03-26_architecture_design.md` |
| 迭代文档 | `时间_文档名称.md` | `2026-03-26_sprint_planning.md`     |
| README   | `README.md`        | `iteration/v1.0/README.md`          |

## 4. 版本整体说明（README.md）

每个迭代版本的 README.md 必须包含：

- 版本目标与范围
- 主要变更内容（与上一版本对比）
- 关键交付物清单
- 版本状态（规划中/进行中/已完成）

## 5. 版本与更新

- P0/P1 文档必须包含版本号和更新日期
- 修改文档必须更新版本号和日期
- 文档末尾添加变更记录

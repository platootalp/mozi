# 代码安全规范（强制）

## 1. 概述

本规范确保代码中无硬编码密钥、敏感信息泄露及安全漏洞。适用于所有代码提交和 PR 审查。

## 2. 密钥与凭证管理

| 要求 | 说明 |
|------|------|
| 禁止硬编码密钥 | API 密钥、密码、令牌必须使用环境变量或密钥管理服务 |
| 禁止提交 secrets | 禁止在代码库中提交任何形式的密钥或凭证 |
| 使用环境变量注入 | 生产环境通过环境变量注入敏感配置 |

## 3. 输入验证与注入防护

| 场景 | 要求 |
|------|------|
| 数据库查询 | 必须使用参数化查询，禁止字符串拼接 SQL |
| 命令执行 | 必须使用列表参数，禁止 shell=True |
| 文件路径 | 必须验证路径规范化，防止路径遍历 |
| 用户输入 | 所有用户输入必须经过验证和清理 |

## 4. 危险函数黑名单

| 函数 | 风险 | 替代方案 |
|------|------|----------|
| eval() | 任意代码执行 | ast.literal_eval() |
| exec() | 任意代码执行 | 特定功能函数 |
| pickle.loads() | 反序列化漏洞 | json.loads() |
| subprocess.shell=True | 命令注入 | 列表参数传递 |
| __import__() | 动态导入风险 | 静态导入 |

## 5. 依赖安全管理

| 检查项 | 频率 | 工具 |
|--------|------|------|
| 已知漏洞扫描 | 每次构建 | pip-audit, npm audit |
| 许可证合规 | 每次发布 | license-checker |
| 硬编码密钥扫描 | 每次 PR | truffleHog |

## 6. 安全审查清单

- [ ] 无硬编码密钥、密码、令牌
- [ ] 所有用户输入经过验证
- [ ] 数据库查询使用参数化
- [ ] 无 eval/exec 危险函数
- [ ] 文件操作路径已验证
- [ ] 新依赖已通过安全审计
- [ ] 敏感操作有日志记录
